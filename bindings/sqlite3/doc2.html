sqlite3

A reusable native binding package for SQLite3.

Importers IMPORT sqlite3 and call its wrapper functions; they never repeat the underlying LINK block. A connection is a Db resource: bind it with RES, and it is closed automatically by lexical drop on every exit path, or explicitly with close.

Types

SqlValue

Union 
EXPORT UNION SqlValue

A dynamically typed SQLite cell value. Result rows surface each cell as one of these members, chosen by the column's runtime type; typed query binders accept the same members to bind a parameter with its true type.

Variants

NameDescription
SqlIntA 64-bit INTEGER cell.
SqlFloatA FLOAT cell.
SqlTextA UTF-8 TEXT cell. A NULL read also surfaces here as "".
SqlBlobA BLOB cell (reserved; faithful binary BLOB reads/binds are not yet supported).
SqlNullA SQL NULL, distinct from an empty string.

Connection

open

Function 
EXPORT FUNC open(path AS String, readOnly AS Boolean) AS Db

Open an EXISTING database and apply a default 5-second busy timeout.

⚠️ Never creates: opening a path that does not yet exist fails rather than silently creating an empty database — call create for that.

Parameters

NameDescription
pathFilesystem path of the database to open.
readOnlyWhen TRUE, open read-only so every write attempt fails. Defaults to FALSE (read/write).

Returns

An open connection handle; ownership moves to the caller.

create

Function 
EXPORT FUNC create(path AS String) AS Db

Create a NEW database (or open it if it already exists) and apply a default 5-second busy timeout.

ℹ️ SQLite has no portable "fail if the file already exists" flag, so create differs from open only in that a missing file is created rather than an error.

Parameters

NameDescription
pathFilesystem path of the database to create or open.

Returns

An open connection handle; ownership moves to the caller.

openOrCreate

Function 
EXPORT FUNC openOrCreate(path AS String) AS Db

Open an existing database OR create one if the path does not yet exist, then apply the default 5-second busy timeout. Use this when "give me a usable handle either way" is what you mean and the open/create distinction does not matter.

Parameters

NameDescription
pathFilesystem path of the database to open or create.

Returns

An open connection handle; ownership moves to the caller.

Execution

execute

Function 
EXPORT FUNC execute(RES db AS Db, sql AS String) AS Nothing

Run one or more SQL statements that produce no result rows — an INSERT, UPDATE, DELETE, PRAGMA, CREATE INDEX, or a ;-separated batch.

ℹ️ Unlike a prepared query (which compiles only the first statement), this executes every statement in sql in order.
🛡️ SQL Injection safe: Values must NOT be concatenated into sql — use the parameterized query overloads for anything containing user input.

Parameters

NameDescription
dbThe open connection (borrowed).
sqlOne or more SQL statements to run.

Errors

CodeDescription
77030008The underlying SQLite call failed; the error carries SQLite's extended code and message.

query

Function 
EXPORT FUNC query(RES db AS Db, sql AS String, params AS List OF String) AS List OF List OF SqlValue

Run a parameterized SQL statement and return its result rows.

ℹ️ Values are NEVER concatenated into the SQL text — they are bound to placeholders, so a value containing quotes or SQL syntax can never alter the statement (no injection).

Four overloads cover the placeholder style and value typing: positional ? or named (:name/@name/$name) placeholders, with parameters supplied either as a list/map of String (bound as text) or of SqlValue (bound with its true SQLite type, so a real NULL, INTEGER, or FLOAT can be bound).

Rows come back as List OF List OF SqlValue — one inner list per row, one SqlValue per column, typed by each column's runtime type. Works for INSERT/UPDATE/DELETE too (the row list is then empty).

Parameters

NameDescription
dbThe open connection (borrowed).
sqlThe SQL to prepare and run, with ? or named placeholders.
paramsThe values to bind, positional (list) or named (map), as text or SqlValue.

Returns

The result rows, each a positional list of SqlValue cells.

Errors

CodeDescription
77030008The underlying SQLite call failed; the error carries SQLite's extended code and message.
Example
LET rows = query(db, "SELECT id, name FROM t WHERE a = ?", ["x"])
query(db, "INSERT INTO t(a, b) VALUES (:a, :b)", Map OF String TO SqlValue { ":a" := SqlInt[7], ":b" := SqlNull[] })

queryMaps

Function 
EXPORT FUNC queryMaps(RES db AS Db, sql AS String, params AS List OF String) AS List OF Map OF String TO SqlValue

Run a parameterized SQL statement and return each row keyed by column name.

ℹ️ Identical to query — same SQL, same parameter binding and overloads, same finalize-on-every-path safety — but each row is a Map OF String TO SqlValue keyed by column name instead of a positional list.

Reach for this when the caller wants row["name"] rather than tracking column order; reach for query when columns may share a name, since a map collapses duplicate keys.

Parameters

NameDescription
dbThe open connection (borrowed).
sqlThe SQL to prepare and run, with ? or named placeholders.
paramsThe values to bind, positional (list) or named (map), as text or SqlValue.

Returns

The result rows, each a map of column name to SqlValue.

Errors

CodeDescription
77030008The underlying SQLite call failed; the error carries SQLite's extended code and message.
Example
LET rows = queryMaps(db, "SELECT id, name FROM t WHERE a = :a", Map OF String TO String { ":a" := "x" })

Schema

createTable

Function 
EXPORT FUNC createTable(RES db AS Db, name AS String, columns AS String) AS Nothing

CREATE TABLE name (columns). Two overloads accept the column definitions either as a raw SQL string or as a column-name -> column-type map.

ℹ️ With a Map OF String TO String, each entry becomes a name type column definition; iteration order is implementation-defined but stable, which is fine since column order in a CREATE TABLE is not significant.

Parameters

NameDescription
dbThe open connection (borrowed).
nameName of the table to create.
columnsColumn definitions: a raw SQL string, or a name -> SQLite-type map.

Errors

CodeDescription
77030008The underlying SQLite call failed; the error carries SQLite's extended code and message.
Example
createTable(db, "users", "id INTEGER PRIMARY KEY, name TEXT")
createTable(db, "items", Map OF String TO String { "id" := "INTEGER", "label" := "TEXT" })

Info

lastInsertRowId

Function 
EXPORT FUNC lastInsertRowId(RES db AS Db) AS Integer

The ROWID assigned by the most recent successful INSERT on db — typically the new INTEGER PRIMARY KEY.

⚠️ Read it immediately after the INSERT, before any other write on the connection.

Parameters

NameDescription
dbThe open connection (borrowed).

Returns

The last inserted ROWID, or 0 if no INSERT has run on the connection.

Returns
