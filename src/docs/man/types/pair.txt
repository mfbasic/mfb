NAME
  pair - the built-in Pair OF A, B two-value product

SYNOPSIS
  Pair OF A, B

DESCRIPTION
  Pair OF A, B is a compiler-owned, always-in-scope generic record. It holds two
  values of possibly different types:

    TYPE Pair OF A, B
      first  AS A
      second AS B
    END TYPE

  Unlike MapEntry, Pair places no comparability constraint on A or B. It is a
  general two-value product, used by collections::zip to pair items from two
  lists position-wise.

  Pair is an ordinary record: it is public, constructible, and copyable when both
  A and B are copyable, so it may be returned, stored in collections, and sent
  across a thread when its members are sendable.

CONSTRUCTION
  Construct a Pair positionally; the type arguments are inferred from the
  expected type:

    LET p AS Pair OF Integer, String = Pair[1, "one"]

  Read the fields with member access:

    LET n AS Integer = p.first
    LET s AS String = p.second

DEFAULTABILITY
  Pair OF A, B is defaultable when both A and B are defaultable. A default Pair
  holds the default value of each member, so a MUT binding without an initializer
  is allowed only when both members are defaultable.

EXAMPLES
  Pair the elements of two lists:

    IMPORT collections
    LET pairs AS List OF Pair OF Integer, String =
      collections::zip([1, 2], ["a", "b"])
    LET first AS Pair OF Integer, String = get(pairs, 0)
