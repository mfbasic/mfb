NAME
  numeric - primitive numeric types and promotion rules

SYNOPSIS
  Integer
  Float
  Fixed
  Money
  Byte

DESCRIPTION
  MFBASIC numeric primitives are checked values. Arithmetic does not silently
  wrap. Failures are ordinary Error/Result failures and route through MATCH,
  TRAP, or caller propagation.

PRIMITIVE TABLE
  Integer
    64-bit signed integer.

  Float
    64-bit IEEE binary floating-point value. MFBASIC does not expose successful
    NaN or infinity results.

  Fixed
    64-bit deterministic binary fixed-point value with a signed 32/32 split.
    The range is approximately -2147483648.0 through 2147483647.9999999998,
    with resolution 1 / 2^32.

  Money
    64-bit exact base-10 fixed-point value scaled to five decimal places, for
    auditable financial amounts. The range is -92233720368547.75808 through
    92233720368547.75807 and every decimal amount within range is represented
    exactly. Money is a DIMENSIONED numeric with a restricted algebra: you may add
    two amounts, scale an amount by a dimensionless number (Integer, Byte, Float,
    Fixed), take the ratio of two amounts (Money / Money is Float), and take a
    remainder (Money MOD Money), but adding a bare number to an amount,
    multiplying two amounts, comparing an amount to a bare number, or raising an
    amount to a power are compile-time errors. Crossing into or out of Money is an
    explicit toMoney / to* call — there is no implicit conversion. Money literals
    use an m or M suffix (1.25m) or an expected Money type. Arithmetic rounding
    follows a runtime mode (money::setRounding); see mfb man money and
    mfb man general toMoney.

  Boolean
    TRUE or FALSE. Boolean is a primitive type, but it is not numeric and does
    not participate in numeric promotion.

  String
    Immutable UTF-8 text. String is a primitive type, but it is not numeric.

  Byte
    Unsigned 8-bit integer with range 0 through 255.

LITERALS
  Numeric literals are initially untyped.

  Integer-looking literals default to Integer when there is no expected type.
  Decimal-looking literals default to Float when there is no expected type.

  When the expected type is Fixed, a decimal literal is rounded to the nearest
  representable Fixed value:

    LET x = 1.25             ' Float
    LET y AS Fixed = 1.25    ' Fixed
    LET z = toFixed("1.25")  ' Fixed, fallible parse

  Integer literals may initialize Byte only when the literal is statically in
  range.

PROMOTION ORDER
  For most mixed numeric operators, the wider type wins:

    Fixed > Float > Integer > Byte

  DIV is different: every valid DIV combination produces Float.

OPERATOR RESULT TABLE
  The result type for +, -, *, ^, /, and MOD is:

    Left       Right      Result
    Byte       Byte       Byte
    Byte       Integer    Integer
    Byte       Fixed      Fixed
    Byte       Float      Float
    Integer    Byte       Integer
    Integer    Integer    Integer
    Integer    Fixed      Fixed
    Integer    Float      Float
    Fixed      Byte       Fixed
    Fixed      Integer    Fixed
    Fixed      Fixed      Fixed
    Fixed      Float      Fixed
    Float      Byte       Float
    Float      Integer    Float
    Float      Fixed      Fixed
    Float      Float      Float

  The result type for DIV is Float for every combination in the table above.

COMPARISONS
  Numeric comparisons use the same operand promotion rules for comparison and
  always return Boolean:

    =  <>  <  >  <=  >=

INTEGER RULES
  Integer arithmetic is checked. Overflow in +, -, *, unary -, exponentiation
  (^), and the minimum-integer MOD -1 case fails with ErrOverflow (77050010).

  Integer operations never wrap.

BYTE RULES
  Byte arithmetic that returns Byte is checked. Results above 255 fail with
  ErrOverflow (77050010). Results below 0 fail with ErrUnderflow (77050011).

  Byte operations never wrap. Runtime conversion to Byte uses toByte and fails
  with ErrOverflow (77050010) when outside 0 through 255.

DIVISION AND MOD
  / uses the operator result table to determine the result type. When / returns
  Byte or Integer, it truncates the quotient toward zero.

  DIV is fractional division and always returns Float.

  MOD uses the operator result table to determine the result type. a MOD b
  fails with ErrInvalidArgument (77050002) when b = 0. Otherwise the remainder
  has the same sign as a, and:

    a = (truncTowardZero(a / b) * b) + (a MOD b)

  Division by zero fails with ErrInvalidArgument (77050002).

EXPONENTIATION
  Integer exponentiation requires a non-negative Integer exponent. Negative
  exponents fail with ErrInvalidArgument (77050002). Overflow fails with
  ErrOverflow (77050010).

FLOAT RULES
  Float uses IEEE 754 binary64 representation, but ordinary source cannot
  construct or import non-finite values successfully.

  Operations or math functions that would produce NaN or infinity fail instead.
  Float domain failures, including division by zero and invalid ^ exponents,
  fail with ErrFloatDomain (77050012). Results that would be NaN fail with
  ErrFloatNaN (77050013). Results that would be infinity fail with ErrFloatInf
  (77050014), except arithmetic overflow to infinity fails with ErrFloatOverflow
  (77050015).

  Imported native Float values that are already NaN or infinity are rejected at
  the boundary with ErrInvalidFormat (77050003).

FIXED RULES
  Fixed is deterministic binary fixed-point arithmetic, not exact decimal
  currency arithmetic. Most decimal fractions are rounded to binary fixed-point
  values.

  Fixed overflow fails with ErrOverflow (77050010). Divide-by-zero and invalid
  numeric domains fail with ErrInvalidArgument (77050002).

CONVERSIONS
  Converting Float or Fixed to Integer or Byte fails with ErrOverflow (77050010)
  when outside the destination range.

  Converting text to a numeric type fails with ErrInvalidFormat (77050003) when
  the text is malformed or names a non-finite value such as NaN or Infinity.
