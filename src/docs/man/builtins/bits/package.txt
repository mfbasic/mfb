NAME
  bits - integer bitwise, shift, and rotate operations

SYNOPSIS
  IMPORT bits
  bits::band(a, b)
  bits::sl(value, count)
  bits::rl32(value, count)
  bits::popCount(value)
  bits::bswap32(value)

DESCRIPTION
  The bits package provides the bitwise integer operations the language operator
  set intentionally omits: the reserved words AND, OR, XOR, and NOT are logical
  (Boolean) operators, so byte-level codecs and other bit-twiddling are written
  with these functions instead. Every operand and result is a raw two's-complement
  64-bit Integer bit pattern; the functions do not interpret sign except where a
  signature says so (sra, the arithmetic right shift). bits is a built-in package:
  IMPORT bits needs no manifest dependency.

  Each function lowers to one (or a few) native AArch64 instructions inline, like
  math::abs, and produces identical results on the native and Binary
  Representation execution paths. All functions are total except the three shift
  operations, which validate their count argument.

  The Boolean ops are named band/bor/bxor/bnot because and/or/xor/not are reserved
  logical-operator keywords and cannot be package member identifiers. The four
  named rotate variants (rl32/rr32/rl64/rr64) map one-to-one to hardware and avoid
  a width parameter: the 32-bit rotates target word-oriented algorithms (such as
  ChaCha20) and the 64-bit rotates target 64-bit arithmetic cores.

FUNCTIONS
  band, bor, bxor, bnot   bitwise AND / OR / XOR / NOT
  sl, sr, sra             logical left / logical right / arithmetic right shift
  rl32, rr32, rl64, rr64  rotate the low 32 bits or all 64 bits left / right
  clz, ctz, popCount      count leading zeros / trailing zeros / set bits
  bswap16, bswap32, bswap64   reverse the bytes of the low 16/32 or all 64 bits

ERRORS
  77050002 (ErrInvalidArgument)
  Raised by sl, sr, and sra when count is outside 0..63.
