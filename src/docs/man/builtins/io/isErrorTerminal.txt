NAME
  isErrorTerminal - test whether standard error is a terminal

SYNOPSIS
  io::isErrorTerminal() AS Boolean

PACKAGE
  io

IMPORTS
  IMPORT io

DESCRIPTION
  Reports whether standard error is connected to an interactive terminal.
  Returns TRUE when standard error is a tty and FALSE when it is redirected
  to a file, a pipe, or any other non-terminal destination.

  Use this to decide whether to emit ANSI colors, progress bars, or other
  terminal-only formatting on the error stream, and to fall back to plain output
  when it is redirected to a file or pipe. The check inspects only the current
  state of standard error (file descriptor 2); it does not modify the stream or
  write any data. To probe the other standard streams instead, use
  io::isOutputTerminal or io::isInputTerminal.

  The function takes no arguments and never fails: a non-terminal error stream
  simply yields FALSE rather than an error.

  Platform note: in app mode the program has no real file descriptors because
  output is rendered by the application window, which is treated as an
  interactive console; in that environment this call returns TRUE.

PARAMETERS
  None.

RETURN VALUE
  AS Boolean
    Returns TRUE when standard error is an interactive terminal and FALSE when
    it is redirected to a file, pipe, or other non-terminal destination. In app
    mode the result is TRUE.

ERRORS
  No errors.

EXAMPLES
  Emit ANSI color on the error stream only when a terminal is attached:

    IF io::isErrorTerminal() THEN
      io::printError("[31mError:[0m something went wrong")
    ELSE
      io::printError("Error: something went wrong")
    END IF

  Capture the current state for later branching:

    LET colorErrors AS Boolean = io::isErrorTerminal()

SEE ALSO
  io::isOutputTerminal, io::isInputTerminal, io::printError, io::flushError
