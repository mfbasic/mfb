NAME
  writeError - write text to standard error without a newline

SYNOPSIS
  io::writeError(value AS String) AS Nothing

PACKAGE
  io

IMPORTS
  IMPORT io

DESCRIPTION
  Writes value to the process standard error stream without appending a
  newline. The value is treated as UTF-8 text and is emitted exactly as stored,
  byte for byte, with no further encoding, escaping, or newline translation. Any
  valid Unicode scalar values are permitted, including multi-byte characters and
  emoji.

  Only String values are accepted. The function performs no implicit conversion,
  so to write a non-string value you must convert it first, for example with
  toString. An empty String writes nothing at all.

  Output is directed to whichever destination is bound to standard error. In a
  normal console program this is file descriptor 2; in app mode the same call is
  routed to the application transcript instead of a file descriptor. The write
  is the function's only side effect. Standard error is written immediately and
  is never buffered, so error output can never sit unseen in a buffer. To append
  a trailing newline, use io::printError; to send text to standard output instead,
  use io::write.

PARAMETERS
  value AS String
    The text to write. Interpreted as UTF-8 and emitted unchanged. May be any
    String, including the empty String and strings containing arbitrary Unicode
    characters.

RETURN VALUE
  AS Nothing
    Returns nothing. The function is invoked for its side effect of writing to
    standard error.

ERRORS
  77020002 (ErrOutput)
  Raised when the underlying write to standard error fails, for example when the
  error stream is closed or a host write error occurs.

EXAMPLES
  Write an error message without a trailing newline:

    io::writeError("failed")

  Write Unicode text including emoji:

    io::writeError("é日😀")

  Convert a non-string value before writing:

    io::writeError(toString(42))

SEE ALSO
  io::printError, io::write, io::flush
