NAME
  flushError - flush standard error

SYNOPSIS
  io::flushError() AS Nothing

PACKAGE
  io

IMPORTS
  IMPORT io

DESCRIPTION
  Drains any pending standard-error data so that everything written so far is
  pushed to the underlying destination. Standard error may be buffered, which
  means text passed to io::writeError or io::printError is not guaranteed to be
  visible to an external reader until the buffer is drained; calling
  io::flushError forces that drain immediately. This is the standard-error
  counterpart of io::flush, which drains standard output.

  The function takes no arguments and has no return value; its only effect is to
  flush the stream. The native runtime writes directly to the host standard-error
  stream and does not maintain an additional MFBASIC error buffer of its own, so
  in practice this requests a flush of whatever buffering the host stream applies.

  Output is directed to whichever destination is bound to standard error. In a
  normal console program this is file descriptor 2; in app mode the same call is
  routed to the application transcript window instead of a file descriptor, where
  writes are already synchronous and the flush completes immediately. When the
  bound stream is not a flushable destination — for example a pipe or terminal
  that reports EINVAL, ENOTSUP, or EOPNOTSUPP — the call is treated as a success
  rather than an error.

PARAMETERS
  None.

RETURN VALUE
  AS Nothing
    Returns nothing. The function is invoked for its side effect of flushing
    standard error.

ERRORS
  77020002 (ErrOutput)
  Raised when the underlying flush of standard error fails, for example when the
  error stream is closed or a host write error occurs.

EXAMPLES
  Force a diagnostic message to appear before continuing:

    io::writeError("failed")
    io::flushError()

  Ensure buffered error output is visible at a checkpoint:

    io::printError("checkpoint reached")
    io::flushError()
