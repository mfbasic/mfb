NAME
  setBuffered - enable or disable standard-output buffering

SYNOPSIS
  io::setBuffered(enabled AS Boolean) AS Nothing

PACKAGE
  io

IMPORTS
  IMPORT io

DESCRIPTION
  Turns opt-in standard-output buffering on or off for the calling thread.
  Buffering is off by default: every io::write and io::print reaches the
  operating system immediately, exactly as it does without this call. After
  io::setBuffered(TRUE), output is instead held in a per-thread buffer and
  written in larger blocks, collapsing a write-heavy loop from one write per call
  into roughly one write per full buffer.

  When buffering is on, held output is drained automatically at the points where
  holding it back would be observable: on io::flush, before any standard-input
  read (io::input, io::readLine, io::readChar, io::readByte) so a buffered prompt
  always appears before the program blocks, when the buffer fills, and at normal
  or signalled program exit. Passing FALSE drains any pending bytes first and then
  disables buffering, so switching buffering off never strands output.

  Buffering is per thread: each thread has its own buffer and enabled flag, and a
  spawned thread starts with buffering off regardless of its parent. Standard
  error is never buffered — io::writeError and io::printError always write
  immediately — so this call affects standard output only.

  Because buffered output is held in memory until drained, a hard crash
  (SIGSEGV, SIGKILL, or an abort) can lose bytes that were written but not yet
  flushed. This is the trade-off buffering accepts in exchange for far fewer
  writes; use io::flush at any point where output must be durable, and leave
  buffering off (the default) when partial-output-on-crash visibility matters.

  Platform note: in app mode standard output is rendered by the application
  window and is never buffered, so this call has no effect.

PARAMETERS
  enabled AS Boolean
    TRUE to enable standard-output buffering, FALSE to drain any pending output
    and disable it.

RETURN VALUE
  AS Nothing
    Returns nothing. The function is invoked for its side effect of changing the
    buffering mode (and, when disabling, draining any pending output).

ERRORS
  No errors. A write failure while draining on the disable transition is not
  reported here; it surfaces from the next io::flush or buffered write instead.

EXAMPLES
  Buffer a write-heavy loop and flush once at the end:

    io::setBuffered(TRUE)
    MUT i AS Integer = 0
    WHILE i < 100000
      io::print(toString(i))
      i = i + 1
    END WHILE
    io::flush()

  Turn buffering back off, draining anything still pending:

    io::setBuffered(FALSE)
