NAME
  toByte - convert an Integer to Byte

SYNOPSIS
  toByte(value AS Integer) AS Byte
  toByte(value AS Money) AS Byte
  toByte(value AS Scalar) AS Byte

PACKAGE
  general

IMPORTS
  None. general functions are always available without an IMPORT statement.

DESCRIPTION
  toByte converts a signed 64-bit Integer to an unsigned 8-bit Byte. A Byte
  holds a whole number in the range 0 through 255 inclusive.

  The Integer is checked against the valid Byte range before conversion. When
  value lies within 0 through 255 it is narrowed to a Byte holding the same
  numeric value. When value is less than 0 or greater than 255 it cannot be
  represented as a Byte and the conversion raises an error rather than
  truncating or wrapping; no modular reduction is performed.

  This is the inverse direction of widening a Byte to Integer with toInt: every
  Byte converts to Integer without loss, while converting back with toByte
  requires the Integer to fall within the Byte range.

OVERLOADS
  toByte(value AS Integer) AS Byte
    Narrows an Integer to Byte, raising an error when value is outside the
    range 0 through 255.

  toByte(value AS Money) AS Byte
    Narrows the whole-unit part of a Money to Byte, raising an error when it is
    outside the range 0 through 255.

  toByte(value AS Scalar) AS Byte
    Narrows a Scalar's code point to Byte, raising an error when the code point
    is greater than 255. The inverse of toScalar(Byte).

PARAMETERS
  value AS Integer
    A signed 64-bit value to narrow to a Byte. It must be in the range 0
    through 255 inclusive; any value outside this range is rejected.

RETURN VALUE
  AS Byte
    The Byte equivalent of value when value is within 0 through 255. The
    returned Byte holds the same numeric value, so toByte(0) yields 0 and
    toByte(255) yields 255.

ERRORS
  77050010 (ErrOverflow)
  Raised when value is less than 0 or greater than 255 and therefore cannot be
  represented as an 8-bit Byte.

TYPE CHECKING
  toByte accepts only Integer, Money, and Scalar values. Calling it with any
  other type is a compile-time error; convert such values to Integer explicitly
  first.

EXAMPLES
  Convert a literal:

    LET value AS Byte = toByte(65)

  Convert an arithmetic result:

    LET original AS Byte = 10
    LET next AS Byte = toByte(toInt(original) + 1)

SEE ALSO
