NAME
  toString - convert supported primitive values and UTF-8 bytes to String

SYNOPSIS
  toString(value AS Integer) AS String
  toString(value AS Float, precision AS Byte = 2) AS String
  toString(value AS Fixed, precision AS Byte = 2) AS String
  toString(value AS Boolean) AS String
  toString(value AS String) AS String
  toString(value AS Byte) AS String
  toString(value AS Scalar) AS String
  toString(value AS List OF Byte) AS String

PACKAGE
  general

IMPORTS
  None. general functions are available without an IMPORT statement.

DESCRIPTION
  toString converts a value from one of the supported built-in types to a
  String. It is intended for explicit text formatting before sending values to
  text-oriented APIs such as io::print, io::write, io::printError, and
  io::writeError.

  Integer and Byte values are rendered as base-10 text; Byte values are unsigned
  and always in the range 0 through 255. Float and Fixed values are rendered as
  decimal text with precision digits after the decimal point, where precision
  defaults to 2 and may also be passed by the name decimals. Fixed is binary
  fixed-point, so its decimal output reflects the representable value, not
  necessarily the original decimal spelling used to create it. A Boolean renders
  as "TRUE" or "FALSE". A String is returned unchanged. A List OF Byte is decoded
  as UTF-8 and the resulting text is returned.

  MFBASIC does not automatically stringify arbitrary values for output. Use
  toString where the language defines an overload, and write an application
  formatting function for records, unions, resources, threads, functions,
  lambdas, or other structured values. toString has no side effects beyond
  producing the result String.

  toString and typeName are diagnostic and formatting conveniences, not security
  boundaries. Do not rely on them to redact secrets or decide whether a value is
  safe to log. Secret-safe output requires explicit application-level formatting
  that omits or redacts sensitive fields.

OVERLOADS
  toString(value AS Integer) AS String
    Converts an Integer to base-10 text.

  toString(value AS Float, precision AS Byte = 2) AS String
    Converts a finite Float to decimal text with precision digits after the
    decimal point. The precision argument may also be passed by the name
    decimals. When omitted, precision defaults to 2.

  toString(value AS Fixed, precision AS Byte = 2) AS String
    Converts a Fixed value to decimal text with precision digits after the
    decimal point. The precision argument may also be passed by the name
    decimals. When omitted, precision defaults to 2. Fixed is binary
    fixed-point, so decimal output reflects the representable value, not
    necessarily the original decimal spelling used to create it.

  toString(value AS Boolean) AS String
    Returns "TRUE" for TRUE and "FALSE" for FALSE.

  toString(value AS String) AS String
    Returns value unchanged.

  toString(value AS Byte) AS String
    Converts a Byte to base-10 text. Byte values are unsigned and are always in
    the range 0 through 255.

  toString(value AS List OF Byte) AS String
    Decodes the byte list as UTF-8 and returns the resulting String. Invalid
    UTF-8 fails with ErrEncoding (77020004).

PARAMETERS
  value AS Integer | Float | Fixed | Boolean | String | Byte | List OF Byte
    The value to convert. Its type selects the overload and determines how the
    result is formatted, as described in DESCRIPTION and OVERLOADS. For the
    List OF Byte overload the bytes are interpreted as a UTF-8 encoded string.

  precision AS Byte = 2
    Present only on the Float and Fixed overloads. The number of digits to emit
    after the decimal point. Defaults to 2 when omitted, and may also be passed
    by the name decimals.

RETURN VALUE
  AS String
    The text representation of value: base-10 digits for Integer and Byte, a
    fixed-precision decimal for Float and Fixed, "TRUE" or "FALSE" for Boolean,
    the original String unchanged, or the UTF-8 decoding of a List OF Byte.

ERRORS
  77020004 (ErrEncoding)
  Raised by the List OF Byte overload when the byte sequence is not valid UTF-8.

  The other overloads do not perform numeric parsing and cannot fail for format
  or range reasons.

TYPE CHECKING
  toString is defined only for the overloads listed in this page. Calling
  toString on user-defined records, unions, enums, resources, threads,
  functions, lambdas, List values other than List OF Byte, or Map values is a
  compile-time type error. toString(Scalar) is infallible and yields the
  one-scalar UTF-8 string for the code point.

  No implicit conversion is performed for unsupported argument types. Convert to
  one of the supported types explicitly, or provide a domain-specific formatter.

EXAMPLES
  Print a number:

    LET count AS Integer = 42
    io::print(toString(count))

  Format a Boolean:

    LET enabled AS Boolean = TRUE
    LET label AS String = "enabled=" & toString(enabled)

  Format a Float with explicit precision:

    LET ratio AS Float = 3.14159
    LET text AS String = toString(ratio, 4)

  Decode UTF-8 bytes:

    LET bytes AS List OF Byte = [104, 101, 108, 108, 111]
    LET text AS String = toString(bytes)
