NAME
  typeName - return the static type display name of a value

SYNOPSIS
  typeName OF T(value AS T) AS String

PACKAGE
  general

IMPORTS
  None. general functions are available without an IMPORT statement.

DESCRIPTION
  typeName returns an implementation-defined display name for the static type of
  value as a String. It is intended for diagnostics, debugging, and
  human-readable messages.

  The name reflects the static type known at compile time, not any runtime tag.
  The argument is used only to determine its static type; its runtime contents
  are never inspected, and typeName has no side effects. For a primitive type the
  name is the type keyword, such as "Integer" or "String". For composite types
  the name spells out the structure, such as "List OF Byte" or
  "Map OF String TO Integer", matching the way the type is written in source.

  typeName is not a serialization or schema API and does not expose runtime
  values. The exact spelling of a name may change between implementations, so
  programs must not parse it or rely on it for security decisions or redaction.
  typeName, toString, and diagnostic messages are not security boundaries;
  secret-safe output requires explicit application-level formatting that omits or
  redacts sensitive fields.

PARAMETERS
  value AS T
    A value of any type T. Only its static type is used to compute the result;
    the value itself is not read.

RETURN VALUE
  AS String
    The implementation-defined display name of the static type T of value, such
    as "Integer", "String", "List OF Byte", or "Map OF String TO Integer".

ERRORS
  No errors.

TYPE CHECKING
  typeName is generic over its single argument and accepts a value of any type T.
  The type parameter T is unconstrained.

EXAMPLES
  Show a primitive type:

    LET name AS String = typeName(42)

  Composite types spell out their structure:

    LET kind AS String = typeName([1, 2, 3])

  Use in diagnostics:

    SUB logType(value AS String)
      io::print("type=" & typeName(value))
    END SUB

SEE ALSO
