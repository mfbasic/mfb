NAME
  accept - accept the next pending connection on a TCP listener

SYNOPSIS
  net::accept(listener AS Listener) AS Socket
  net::accept(listener AS Listener, timeoutMs AS Integer) AS Socket

PACKAGE
  net

IMPORTS
  IMPORT net

DESCRIPTION
  accept removes the next pending connection from a Listener's queue and returns
  a connected Socket for communicating with that client. The listener must have
  been placed in the listening state by net::listenTcp and must still be open.
  Each call accepts a single connection; call accept in a loop to serve clients
  as they arrive.

  The optional timeoutMs bounds how long accept waits for a pending connection,
  in milliseconds. When timeoutMs is positive and no client arrives before the
  deadline, accept fails with ErrTimeout. When omitted, zero, or negative, accept
  blocks until a client connects.

  The returned Socket is independent of the listener: it stays usable after the
  listener is closed, and closing it does not affect the listener. Socket
  resources are closed by lexical drop at scope exit or explicitly with
  net::close. Read and write data with net::read, net::readText, net::write, and
  net::writeText, and inspect the endpoints with net::localAddress and
  net::remoteAddress.

OVERLOADS
  net::accept(listener AS Listener) AS Socket
    Blocks until a client connects and returns the connected Socket.

  net::accept(listener AS Listener, timeoutMs AS Integer) AS Socket
    Waits at most timeoutMs for a pending connection, then fails with ErrTimeout
    if none arrived (a non-positive timeoutMs blocks indefinitely instead).

PARAMETERS
  listener AS Listener
    An open Listener resource in the listening state, as returned by
    net::listenTcp. The listener is not consumed and remains available for
    further accept calls.

  timeoutMs AS Integer
    Optional. The maximum time, in milliseconds, to wait for a pending
    connection. A positive value that elapses with no connection fails with
    ErrTimeout. When omitted, zero, or negative, accept blocks indefinitely
    until a client connects.

RETURN VALUE
  AS Socket
    A connected Socket for communicating with the accepted client. The Socket is
    independent of the listener and is closed by lexical drop at scope exit
    unless closed earlier with net::close.

ERRORS
  77050008 (ErrTimeout)
  Raised when timeoutMs is positive and no connection arrives before the deadline
  elapses. With timeoutMs omitted, zero, or negative, accept blocks indefinitely
  and never raises this error.

  77070003 (ErrNetworkFailed)
  Raised when the underlying accept call fails and a connection could not be
  accepted from the listener's queue.

  77030004 (ErrResourceClosed)
  Raised when the listener has already been closed.

  77010001 (ErrOutOfMemory)
  Raised when memory for the connected Socket handle could not be allocated.

EXAMPLES
  Accept a single client and read a request line:

    IMPORT net

    RES listener = net::listenTcp("0.0.0.0", 8080)
    RES client = net::accept(listener)
    LET request = net::readText(client, 1024)
    ' client and listener are closed by lexical drop when this scope ends

  Serve clients in a loop:

    IMPORT net

    RES listener = net::listenTcp("0.0.0.0", 8080)
    DO
      RES client = net::accept(listener)
      net::writeText(client, "hello\n")
      ' client is closed by lexical drop at the end of each iteration
    LOOP
