NAME
  respondFile - serve an open file as a 200 response

SYNOPSIS
  http::respondFile(file AS RES File) AS Response
  http::respondFile(file AS RES File, contentType AS String) AS Response

PACKAGE
  http

IMPORTS
  IMPORT fs
  IMPORT http

DESCRIPTION
  respondFile reads an open fs::File fully into the body of a 200 http::Response and
  closes the file. It takes ownership of the file (RES File), so the handle is
  closed by lexical drop even on an error path — the RES is consumed and must not be
  used afterward. When contentType is "" (the default), Content-Type defaults to
  "application/octet-stream".

  respondFile is the low-level static primitive; most handlers call http::respondPath,
  which resolves a request path safely under a root, infers the Content-Type from
  the extension, and calls respondFile. The whole file is buffered into the response
  body — acceptable for a single-threaded dev/static server; serving a large file
  blocks the server until it finishes.

ERRORS
  Read failures are propagated from fs.

EXAMPLES
  Serving a specific file with an explicit type:

    RES f AS File = fs::openFile("./page.html")
    RETURN http::respondFile(f, "text/html; charset=utf-8")

SEE ALSO
  http::respondPath, fs::openFile, fs::readAllBytes
