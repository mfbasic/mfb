NAME
  respondPath - serve a request path safely under a root directory

SYNOPSIS
  http::respondPath(req AS Request, root AS String) AS Response

PACKAGE
  http

IMPORTS
  IMPORT http

DESCRIPTION
  respondPath resolves the requested file under root and serves it. The relative
  path is req.params["*"] when the matched route captured a wildcard remainder,
  otherwise req.path; a leading "/" is stripped and an empty path resolves to
  "index.html". The Content-Type is inferred from the file extension (html, css,
  js, json, png, jpeg, svg, woff2, wasm, and others; application/octet-stream
  otherwise), the whole file is buffered into the body, and a 200 is returned.

  Path-traversal safety is built in. respondPath confines the resolved path to
  root before opening it: the candidate is canonicalized with the host realpath
  resolution (following symlinks and collapsing "..") and confirmed to be within
  root. A missing file yields 404; any path that escapes root (via "..", an
  absolute path, or a symlink out) yields 403 and is never opened. This is the one
  security-critical behavior of the static helpers and is not left to the caller.

ERRORS
  respondPath returns 404 for a missing file and 403 for a path that escapes root;
  it does not fail on either. Read failures on a confined, existing file are
  propagated from fs.

EXAMPLES
  A catch-all static route:

    FUNC serveStatic(req AS http::Request) AS http::Response
      RETURN http::respondPath(req, "./public")
    END FUNC

    ' registered as:  http::route("/static/*", serveStatic)
