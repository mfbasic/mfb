NAME
  withHeader - copy a response with one header set

SYNOPSIS
  http::withHeader(resp AS Response, name AS String, value AS String) AS Response

PACKAGE
  http

IMPORTS
  IMPORT http

DESCRIPTION
  withHeader returns a copy of resp with the header name set to value — sugar over
  WITH resp { headers := ... }. Response header names are sent as written (unlike
  request header names, which are lowercased on parse). Setting an existing header
  replaces its value.

  The reserved framing headers are still server-controlled on emit: a Content-Length
  or Connection header set here is ignored so handleRequest can supply the correct
  framing.

EXAMPLES
  Adding a caching header to an ok response:

    LET resp = http::withHeader(http::ok("body"), "Cache-Control", "no-store")

SEE ALSO
  http::ok, http::responseDefault, http::status
