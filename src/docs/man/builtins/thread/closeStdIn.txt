NAME
  closeStdIn - unsubscribe a thread from the stdin broadcast log

SYNOPSIS
  thread::closeStdIn() AS Nothing
  thread::closeStdIn(t AS Thread OF Msg TO Out) AS Nothing

PACKAGE
  thread

IMPORTS
  IMPORT thread

DESCRIPTION
  Unsubscribes a thread from the process-global stdin broadcast log and releases
  its hold on the log. The no-argument form unsubscribes the calling thread; the
  one-argument form takes a parent Thread handle t and unsubscribes the worker
  behind it.

  Unsubscribing removes the thread's cursor from the log's subscriber registry and
  recomputes the reclamation point, so the log can free blocks that no remaining
  subscriber still needs. A subscriber that had stalled and was applying
  backpressure to the reader stops doing so once it unsubscribes, unblocking a
  producer that was waiting on the memory cap.

  closeStdIn is a no-op if the thread is not subscribed or the log was never set
  up, so it is safe to call unconditionally. Thread teardown auto-unsubscribes, so
  an exited or crashed thread never permanently pins the log's reclamation point;
  the explicit call is for releasing stdin earlier, while the thread keeps running.

  After closeStdIn a thread that reads stdin again raises ErrInvalidContext until
  it resubscribes with thread::openStdIn.

PARAMETERS
  t AS Thread OF Msg TO Out
    Optional. A live parent Thread handle whose worker should be unsubscribed. When
    omitted, the calling thread is unsubscribed. Msg is the worker's inbound
    message type and Out its output type; closeStdIn reads neither.

RETURN VALUE
  AS Nothing
    closeStdIn returns no value. On return the target thread is not subscribed.

EXAMPLES
  Release stdin from the calling thread when done reading:

    thread::openStdIn()
    LET line AS String = io::readLine()
    thread::closeStdIn()

SEE ALSO
  thread::openStdIn, thread::start, io::readLine
