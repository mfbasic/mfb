NAME
  openStdIn - subscribe a thread to the stdin broadcast log

SYNOPSIS
  thread::openStdIn() AS Nothing
  thread::openStdIn(t AS Thread OF Msg TO Out) AS Nothing

PACKAGE
  thread

IMPORTS
  IMPORT thread

DESCRIPTION
  Subscribes a thread to the process-global stdin broadcast log so that it can
  read standard input. The runtime owns file descriptor 0 and reads it in chunks
  into one append-only log; every subscribed thread reads its own independent
  cursor over that log, so each subscriber sees the entire stdin byte stream from
  the point it subscribed and a byte read by one subscriber is never consumed out
  from under another.

  The no-argument form subscribes the calling thread. The one-argument form takes
  a parent Thread handle t and subscribes the worker behind it; use it to grant a
  worker access to stdin before the worker begins reading. A subscriber always
  joins at the current frontier — the next byte the runtime has yet to read — and
  sees every byte that arrives after. There is no "replay from the start" form.

  A single-threaded program never needs to call openStdIn: the compiler inserts an
  implicit main-thread subscription at program entry (before any input is read),
  so main's cursor starts at offset 0 and it sees the whole stream. openStdIn is
  idempotent — subscribing an already-subscribed thread is a no-op — so the
  explicit call is harmless and documents intent.

  A thread that reads stdin (io::readLine, io::input, io::readChar, io::readByte)
  without a subscription raises ErrInvalidContext rather than silently seeing an
  empty stream. Reclamation is bounded: the log's memory is bounded by a
  build-time high-water mark, and a stalled subscriber applies backpressure to the
  reader rather than growing memory without limit.

PARAMETERS
  t AS Thread OF Msg TO Out
    Optional. A live parent Thread handle whose worker should be subscribed. When
    omitted, the calling thread is subscribed. Msg is the worker's inbound message
    type and Out its output type; openStdIn reads neither.

RETURN VALUE
  AS Nothing
    openStdIn returns no value. On return the target thread is subscribed at the
    current stream frontier.

EXAMPLES
  Subscribe the calling thread explicitly:

    thread::openStdIn()
    LET line AS String = io::readLine()

  Grant a worker access to stdin before it reads:

    LET w AS Thread OF String TO Integer = thread::start(workers::countLines, "", 1, 1)
    thread::openStdIn(w)
