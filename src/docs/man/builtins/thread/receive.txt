NAME
  receive - read a message from a thread handle

SYNOPSIS
  thread::receive(t AS Thread OF Msg TO Out) AS Msg
  thread::receive(t AS Thread OF Msg TO Out, timeoutMs AS Integer) AS Msg
  thread::receive(t AS ThreadWorker OF Msg TO Out) AS Msg
  thread::receive(t AS ThreadWorker OF Msg TO Out, timeoutMs AS Integer) AS Msg

PACKAGE
  thread

IMPORTS
  IMPORT thread

DESCRIPTION
  Dequeues one message from the bounded data-plane queue selected by the handle
  type, returning it by move. A parent Thread handle reads from the worker's
  outbound queue, taking a message the worker produced; a ThreadWorker handle
  reads from the worker's inbound queue, taking a message the parent sent. The
  matching writer is thread::send on the opposite handle. receive operates on the
  data plane only and carries values; resources move on the separate resource
  plane and are taken with thread::accept.

  The dequeued value has the thread's message type Msg (a thread whose message
  type is Unknown yields a value of type Unknown). The value is removed from the
  queue and moved into the calling package instance; nothing is shared between
  the two threads, and dequeuing frees a slot so a sender blocked on a full queue
  can proceed.

  There are two forms, and both behave the same way whether the handle is a
  parent Thread or a ThreadWorker. The no-timeout form waits: receive blocks until
  a message arrives, the queue is closed, or the worker is cancelled. The
  timeoutMs form does not have a default and controls a bounded wait when the
  queue is currently empty: timeoutMs must be zero or positive. With timeoutMs = 0
  the call does not wait and fails at once with ErrNotFound if no message is
  queued; a positive timeoutMs waits up to that many milliseconds, failing with
  ErrTimeout if the timeout elapses while the queue is still empty. A negative
  timeoutMs is rejected with ErrInvalidArgument -- to wait indefinitely, omit the
  argument.

  receive participates in cooperative cancellation and observes queue closure. On
  a parent Thread handle, if the worker has completed and left its outbound queue
  empty, or the outbound queue has been closed, the call fails with ErrNotFound
  (so a blocking parent receive wakes when the worker exits rather than waiting
  forever); if the parent Thread handle itself has already been closed (for
  example after its result was retrieved with thread::waitFor or t.result) the
  call fails with ErrResourceClosed. On a ThreadWorker handle, a receive that is
  waiting or finds the queue empty fails with ErrInterrupted when cancellation of
  the worker has been requested, and with ErrNotFound when the inbound queue has
  been closed.

OVERLOADS
  thread::receive(t AS Thread OF Msg TO Out) AS Msg
    Blocking parent-side receive. Reads from the worker's outbound queue, waiting
    until a message arrives, the worker completes with an empty outbound queue
    (ErrNotFound), or the queue is closed (ErrNotFound). Fails with
    ErrResourceClosed when the parent Thread handle is already closed.

  thread::receive(t AS Thread OF Msg TO Out, timeoutMs AS Integer) AS Msg
    Timed parent-side receive. timeoutMs must be >= 0: 0 polls once and a positive
    value waits up to that many milliseconds. A negative timeoutMs is rejected with
    ErrInvalidArgument.

  thread::receive(t AS ThreadWorker OF Msg TO Out) AS Msg
    Blocking worker-side receive. Reads from the worker's inbound queue, waiting
    until a message arrives, cancellation is requested (ErrInterrupted), or the
    inbound queue is closed (ErrNotFound).

  thread::receive(t AS ThreadWorker OF Msg TO Out, timeoutMs AS Integer) AS Msg
    Timed worker-side receive. timeoutMs must be >= 0: 0 polls once and a positive
    value waits up to that many milliseconds. A negative timeoutMs is rejected with
    ErrInvalidArgument.

PARAMETERS
  t AS Thread OF Msg TO Out | ThreadWorker OF Msg TO Out
    The thread handle whose data-plane queue is read. A parent Thread handle reads
    the worker's outbound queue; a ThreadWorker handle reads the worker's inbound
    queue. Msg is the thread's message type and Out its output type.

  timeoutMs AS Integer
    Optional. When omitted, receive blocks until a message arrives, the queue
    closes, or the worker is cancelled. When given, it is the maximum time in
    milliseconds to wait for a message and must be >= 0: 0 does not wait and fails
    immediately with ErrNotFound if no message is queued, and a positive value
    waits up to that many milliseconds before failing with ErrTimeout. A negative
    timeoutMs is rejected with ErrInvalidArgument.

RETURN VALUE
  AS Msg
    The dequeued message, of the thread's message type Msg, removed from the queue
    and moved to the caller. A handle typed with message Unknown yields a value of
    type Unknown.

ERRORS
  77050002 (ErrInvalidArgument)
  Raised when an explicit timeoutMs is negative. Omit the argument to wait
  indefinitely.

  77050004 (ErrNotFound)
  Raised when no message is available without waiting (timeoutMs = 0 and the queue
  is empty), when the queue has been closed, or when a parent Thread handle's
  worker has completed with an empty outbound queue.

  77050008 (ErrTimeout)
  Raised when the queue is empty and no message arrives before a positive
  timeoutMs elapses.

  77050009 (ErrInterrupted)
  Raised by the ThreadWorker overload when a waiting or empty receive observes
  that cancellation of the worker has been requested.

  77030004 (ErrResourceClosed)
  Raised by the parent Thread overload when the Thread handle has already been
  closed, such as after its result was retrieved with thread::waitFor or t.result.

TYPE CHECKING
  t must be a Thread or ThreadWorker handle; the queue read is selected by which
  of the two it is, and the result type is the handle's message type Msg (Unknown
  for a handle whose message type is Unknown). timeoutMs, when supplied, must be an
  Integer.

EXAMPLES
  Block until a message arrives (the common "wait for work" form):

    LET work AS String = thread::receive(t)

  Take a message without waiting, failing with ErrNotFound if none is queued:

    LET message AS String = thread::receive(t, 0)

  Wait up to 10 milliseconds for a message:

    LET timed AS String = thread::receive(t, 10)
