NAME
  types - the crypto package record types

SYNOPSIS
  crypto::Sealed
  crypto::KeyPair

PACKAGE
  crypto

IMPORTS
  IMPORT crypto

DESCRIPTION
  The crypto package defines two record types. crypto::Sealed is the output of
  authenticated encryption (AEAD): it pairs a ciphertext with its authentication
  tag. crypto::KeyPair is the output of public-key key generation: it pairs a
  private key with its matching public key. Both are plain records; their fields
  are List OF Byte and are read with ordinary field access.

  Sealed is produced by crypto::aes256GcmSeal and crypto::chacha20Poly1305Seal,
  and its two fields are passed back to the corresponding open operation
  (crypto::aes256GcmOpen, crypto::chacha20Poly1305Open) to verify and decrypt.

  KeyPair is produced by crypto::generateEd25519 and by the NIST-curve
  generators crypto::generateP256, crypto::generateP384, and crypto::generateP521.
  Its privateKey feeds the signing operations and its publicKey feeds
  verification.

  crypto::Sealed
    The result of AEAD sealing: an authenticated ciphertext.

    Field         Type            Description
    ciphertext    List OF Byte    The encrypted bytes. The same length as the
                                  original plaintext.
    tag           List OF Byte    The authentication tag: 16 bytes (AES-256-GCM
                                  or ChaCha20-Poly1305). Binds the ciphertext and
                                  any additional authenticated data; checked in
                                  constant time by the open operation.

  crypto::KeyPair
    A generated public/private key pair. The byte encodings are identical on
    every target and are wire-compatible with OpenSSL and pyca.

    Field         Type            Description
    privateKey    List OF Byte    The private key. Secret. See the warning below.
    publicKey     List OF Byte    The public key. Safe to publish and distribute.

    Encodings. For the NIST curves (P-256/384/521), the fields use the
    self-contained SEC1 uncompressed encodings agreed by the spec:

      privateKey = 0x04 || X || Y || K
        the uncompressed point 0x04 || X || Y followed by the big-endian scalar K
        (97 bytes for P-256, 145 for P-384, 199 for P-521).

      publicKey  = 0x04 || X || Y
        the uncompressed SEC1 point (65 / 97 / 133 bytes for P-256 / P-384 /
        P-521).

    For Ed25519, privateKey is the 32-byte seed and publicKey is the 32-byte
    public key.

  Secret safety. crypto::KeyPair.privateKey holds secret key material. Never log,
  print, or serialize it in diagnostics: typeName, toString, and error messages
  are not security boundaries. Keep the private key confidential and distribute
  only publicKey.

RETURN VALUE
  Not applicable. These are record types returned by the crypto functions listed
  above.

ERRORS
  None. The types themselves raise no errors; the functions that produce and
  consume them do.

EXAMPLES
  Read the fields of a Sealed record:

    IMPORT crypto
    IMPORT encoding

    LET box AS crypto::Sealed = crypto::aes256GcmSeal(key, nonce, plaintext)
    PRINT encoding::hexEncode(box.ciphertext)
    PRINT encoding::hexEncode(box.tag)

  Generate a KeyPair and publish only the public key:

    IMPORT crypto

    LET kp AS crypto::KeyPair = crypto::generateEd25519()
    LET sig AS List OF Byte = crypto::ed25519Sign(kp.privateKey, message)
    ' Distribute kp.publicKey; keep kp.privateKey secret.
