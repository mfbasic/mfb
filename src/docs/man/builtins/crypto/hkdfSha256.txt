NAME
  hkdfSha256 - HKDF key derivation over HMAC-SHA-256

SYNOPSIS
  crypto::hkdfSha256(ikm AS List OF Byte, salt AS List OF Byte, info AS List OF Byte, length AS Integer) AS List OF Byte

PACKAGE
  crypto

IMPORTS
  IMPORT crypto

DESCRIPTION
  crypto::hkdfSha256 is the HKDF key-derivation function of RFC 5869 instantiated
  over HMAC-SHA-256. It turns input keying material of arbitrary quality into one
  or more cryptographically strong keys of a chosen length. HKDF runs in two
  phases: an extract step folds ikm and salt into a fixed-length pseudorandom key
  (the HMAC output, 32 bytes for SHA-256), and an expand step stretches that key
  into length output bytes bound to the info context string.

  salt is optional in the RFC sense: passing an empty list selects the default
  all-zero salt of one hash block. info may also be empty; when non-empty it
  domain-separates derived keys so the same ikm can safely produce independent
  keys for different purposes. The maximum output length is 255 * 32 = 8160 bytes,
  the ceiling imposed by HKDF-Expand's single-byte block counter.

  The function is pure and total within its length bound: the same four arguments
  always yield the same bytes, on every target. HKDF is designed for high-entropy
  ikm (for example a Diffie-Hellman shared secret); to derive keys from a
  low-entropy password, use crypto::pbkdf2Sha256 instead.

PARAMETERS
  ikm AS List OF Byte
    The input keying material to derive from. May be any length, including empty.

  salt AS List OF Byte
    A non-secret salt. Pass an empty list to use HKDF's default all-zero salt.

  info AS List OF Byte
    Optional context and application-specific information for domain separation.
    May be empty.

  length AS Integer
    The number of output bytes to produce. Must be in 0 .. 8160 (255 * 32).

RETURN VALUE
  AS List OF Byte
    Exactly length pseudorandom bytes of derived key material.

ERRORS
  77050002 (ErrInvalidArgument)
  Raised when length is negative or exceeds 255 * 32 (8160), the maximum output
  HKDF-Expand can produce for a 32-byte hash.

EXAMPLES
  Derive a 32-byte key from a shared secret:

    IMPORT crypto

    LET key AS List OF Byte = crypto::hkdfSha256(secret, salt, info, 32)

  An empty salt selects the RFC default:

    IMPORT crypto

    LET empty AS List OF Byte = collections::listOfByte()
    LET key AS List OF Byte = crypto::hkdfSha256(secret, empty, empty, 64)
