NAME
  aes256GcmOpen - AES-256-GCM authenticated decryption (NIST SP 800-38D)

SYNOPSIS
  crypto::aes256GcmOpen(key AS List OF Byte, nonce AS List OF Byte, ciphertext AS List OF Byte, tag AS List OF Byte) AS List OF Byte
  crypto::aes256GcmOpen(key AS List OF Byte, nonce AS List OF Byte, ciphertext AS List OF Byte, tag AS List OF Byte, aad AS List OF Byte) AS List OF Byte

PACKAGE
  crypto

IMPORTS
  IMPORT crypto

DESCRIPTION
  crypto::aes256GcmOpen verifies and decrypts a ciphertext produced by
  crypto::aes256GcmSeal, using AES-256 in Galois/Counter Mode as specified by
  NIST SP 800-38D. It recomputes the authentication tag over the ciphertext and
  the additional authenticated data, compares it against tag in constant time,
  and returns the recovered plaintext only if they match.

  The function fails closed. On any tag mismatch it raises
  ErrAuthenticationFailed and returns no plaintext at all — not the partial or
  unverified decryption. A mismatch means the ciphertext, tag, nonce, or aad was
  altered, truncated, or does not belong to this key; the message must be
  rejected. Verification is not optional, and the tag comparison is
  content-independent in time so it does not leak how much of the tag matched.

  key must be exactly 32 bytes (a 256-bit key) and nonce must be exactly 12 bytes
  (the standard 96-bit GCM nonce). To open successfully, key, nonce, ciphertext,
  tag, and aad must all be identical to those from the sealing call: the aad is
  authenticated but not carried in the ciphertext, so the same aad must be
  supplied here. aad defaults to the empty list when omitted.

  The cipher is a portable software core computed over the bits package, so its
  behavior is byte-identical on every target (macOS/Linux, aarch64/x86-64).

PARAMETERS
  key AS List OF Byte
    The 256-bit AES key. Must be exactly 32 bytes and identical to the sealing
    key. This value is secret.

  nonce AS List OF Byte
    The 96-bit GCM nonce. Must be exactly 12 bytes and identical to the nonce
    used to seal the message.

  ciphertext AS List OF Byte
    The encrypted bytes, from crypto::Sealed.ciphertext.

  tag AS List OF Byte
    The 16-byte authentication tag, from crypto::Sealed.tag.

  aad AS List OF Byte
    Optional additional authenticated data. Must be identical to the aad passed
    when sealing. Defaults to the empty list.

RETURN VALUE
  AS List OF Byte
    The recovered plaintext, the same length as the original message. Returned
    only when the tag verifies.

ERRORS
  77050016 (ErrAuthenticationFailed)
  Raised when the authentication tag does not verify — the ciphertext, tag,
  nonce, or aad was altered or does not belong to this key. No plaintext is
  returned.

  77050002 (ErrInvalidArgument)
  Raised when key is not exactly 32 bytes, or when nonce is not exactly 12 bytes.

EXAMPLES
  Round-trip: seal then open:

    IMPORT crypto

    LET key AS List OF Byte = crypto::randomBytes(32)
    LET nonce AS List OF Byte = crypto::randomBytes(12)
    LET box AS crypto::Sealed = crypto::aes256GcmSeal(key, nonce, plaintext)
    LET clear AS List OF Byte = crypto::aes256GcmOpen(key, nonce, box.ciphertext, box.tag)

  A tampered ciphertext or tag raises ErrAuthenticationFailed and returns nothing:

    IMPORT crypto

    ' If box.tag has been altered in transit, this call fails closed:
    LET clear AS List OF Byte = crypto::aes256GcmOpen(key, nonce, box.ciphertext, box.tag)
