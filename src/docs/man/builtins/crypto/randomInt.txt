NAME
  randomInt - cryptographically secure uniform random integer in a range

SYNOPSIS
  crypto::randomInt(min AS Integer, max AS Integer) AS Integer

PACKAGE
  crypto

IMPORTS
  IMPORT crypto

DESCRIPTION
  crypto::randomInt returns a uniformly distributed random Integer in the
  inclusive range [min, max]. Both endpoints are attainable, so the number of
  possible results is max - min + 1. The randomness comes from the same OS CSPRNG
  as crypto::randomBytes (getentropy), so results are cryptographically secure and
  not seedable or reproducible across runs.

  The distribution is unbiased. Rather than reducing raw entropy modulo the range
  (which skews toward smaller values when the range does not divide the entropy
  space evenly), randomInt uses rejection sampling: it draws fresh bytes and
  discards any draw that would fall outside the largest exact multiple of the
  range, guaranteeing every value in [min, max] is equally likely.

  This is the cryptographic counterpart to math::rand's integer helpers (see mfb
  man math rand), which are fast and seedable but NOT cryptographically secure.
  Use crypto::randomInt whenever the value must be unpredictable to an adversary.

PARAMETERS
  min AS Integer
    The inclusive lower bound of the range.

  max AS Integer
    The inclusive upper bound of the range. Must be greater than or equal to min.

RETURN VALUE
  AS Integer
    A uniformly distributed value x with min <= x <= max.

ERRORS
  77050002 (ErrInvalidArgument)
  Raised when min is greater than max, or when the range max - min + 1 is too
  large to sample.

EXAMPLES
  Roll a fair six-sided die:

    IMPORT crypto

    LET roll AS Integer = crypto::randomInt(1, 6)

  A single-value range always returns that value:

    IMPORT crypto

    LET x AS Integer = crypto::randomInt(42, 42)

SEE ALSO
