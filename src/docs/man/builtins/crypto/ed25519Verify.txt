NAME
  ed25519Verify - verify an Ed25519 signature (RFC 8032)

SYNOPSIS
  crypto::ed25519Verify(publicKey AS List OF Byte, message AS List OF Byte, signature AS List OF Byte) AS Boolean

PACKAGE
  crypto

IMPORTS
  IMPORT crypto

DESCRIPTION
  crypto::ed25519Verify checks whether signature is a valid Ed25519 signature of
  message under publicKey, following RFC 8032 PureEdDSA. It returns TRUE if and
  only if the signature verifies for that exact key and message, and FALSE
  otherwise. Ed25519 signing is deterministic, but verification depends only on
  the three inputs and is what this function performs.

  publicKey is the 32-byte Ed25519 public key. message is the raw message bytes
  that were signed (PureEdDSA hashes the whole message internally; no pre-hashing
  is applied). signature is the 64-byte Ed25519 signature produced by
  crypto::ed25519Sign or an interoperating implementation.

  Verification is total and platform-independent: the same (publicKey, message,
  signature) triple yields the same verdict on macOS and Linux, on aarch64 and
  x86-64, and matches other conforming Ed25519 implementations (signatures
  interoperate across platforms and with standard toolkits). A signature that
  does not match is reported as FALSE, not as an error.

  A wrong verdict is distinguished from a malformed key. If publicKey is not
  exactly 32 bytes it cannot name an Ed25519 key at all, so the call raises
  ErrInvalidArgument rather than returning a verdict. A valid-length key with a
  signature that simply does not verify returns FALSE.

PARAMETERS
  publicKey AS List OF Byte
    The Ed25519 public key. Must be exactly 32 bytes. Any other length is an
    error, not a false verdict.

  message AS List OF Byte
    The message bytes whose signature is being verified. Any length is accepted,
    including empty.

  signature AS List OF Byte
    The candidate 64-byte Ed25519 signature.

RETURN VALUE
  AS Boolean
    TRUE if signature is a valid Ed25519 signature of message under publicKey;
    FALSE if it is not. A FALSE result is a normal outcome, not an error.

ERRORS
  77050002 (ErrInvalidArgument)
    Raised when publicKey is not exactly 32 bytes. A valid-length key paired with
    a non-matching signature does not raise this error; it returns FALSE.

EXAMPLES
  Verify a signature produced by ed25519Sign:

    IMPORT crypto

    LET pair AS KeyPair = crypto::generateEd25519()
    LET sig AS List OF Byte = crypto::ed25519Sign(pair.privateKey, message)
    LET ok AS Boolean = crypto::ed25519Verify(pair.publicKey, message, sig)

  A tampered message fails verification (returns FALSE, not an error):

    IMPORT crypto

    LET ok AS Boolean = crypto::ed25519Verify(pair.publicKey, altered, sig)
