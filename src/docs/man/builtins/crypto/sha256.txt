NAME
  sha256 - SHA-256 cryptographic hash (FIPS 180-4)

SYNOPSIS
  crypto::sha256(data AS List OF Byte) AS List OF Byte
  crypto::sha256(data AS String) AS List OF Byte

PACKAGE
  crypto

IMPORTS
  IMPORT crypto

DESCRIPTION
  crypto::sha256 computes the SHA-256 message digest of data, as specified by
  FIPS 180-4. The result is a fixed 32-byte (256-bit) digest returned as a
  List OF Byte. The function is total: any input produces a digest and it never
  raises an error.

  The hash is a portable software core computed over the bits package, so its
  output is byte-identical on every target (macOS/Linux, aarch64/x86-64).

  Overloads. Two forms are provided:

    - List OF Byte form hashes the raw bytes exactly as given.
    - String form hashes the UTF-8 encoding of the string. It is equivalent to
      first converting the string to its UTF-8 bytes and hashing those.

  A digest is raw binary. To display or store it as text, stringify it with
  encoding::hexEncode (lowercase hex) or encoding::base64Encode.

PARAMETERS
  data AS List OF Byte
    The bytes to hash.

  data AS String
    A string whose UTF-8 bytes are hashed.

RETURN VALUE
  AS List OF Byte
    The 32-byte SHA-256 digest.

ERRORS
  None.

EXAMPLES
  Hash a byte list and print it as hex:

    IMPORT crypto
    IMPORT encoding

    LET digest AS List OF Byte = crypto::sha256(raw)
    PRINT encoding::hexEncode(digest)

  Hash a string (its UTF-8 bytes):

    IMPORT crypto
    IMPORT encoding

    PRINT encoding::hexEncode(crypto::sha256("hello"))
