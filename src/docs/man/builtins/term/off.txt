NAME
  off - leave terminal TUI mode and restore the terminal to its prior state

SYNOPSIS
  term::off() AS Nothing

PACKAGE
  term

IMPORTS
  IMPORT term

DESCRIPTION
  Switches the terminal out of TUI mode, the full-screen drawing mode entered by
  term::on. After term::off returns, the term:: surface is no longer active: every
  term:: call except term::on and term::isOn becomes a no-op again, so cursor
  movement, color, attribute, and clear calls have no effect until term::on is
  called once more.

  On the console backend the function tears down the drawing surface and restores
  the host terminal to the state it had before term::on: it leaves the alternate
  screen buffer, so the user's previous shell contents reappear, makes the cursor
  visible, and clears any term:: text attributes so subsequent ordinary output is
  drawn with the terminal's normal colors and styling. In app mode the same call
  deactivates the application's terminal view instead of writing to a file
  descriptor.

  The function takes no arguments and has no return value; its only effect is to
  deactivate TUI mode and restore the terminal. Calling term::off while TUI mode
  is already off is allowed and does nothing: because the module is gated, an
  off call has no effect unless TUI mode is currently on. After term::off returns,
  term::isOn reports false until term::on is called again.

PARAMETERS
  None.

RETURN VALUE
  AS Nothing
    Returns nothing. The function is invoked for its side effect of leaving TUI
    mode and restoring the terminal to its pre-TUI state.

ERRORS
  No errors.

EXAMPLES
  Enter TUI mode, draw, then restore the terminal on exit:

    IMPORT term
    IMPORT io

    term::on()
    term::moveTo(0, 0)
    term::setForeground(255, 0, 0)
    io::print("hello in red")
    term::off()

  term::off is a no-op when TUI mode is already off:

    IF term::isOn() THEN
      term::off()
    END IF

SEE ALSO
  term::on, term::isOn, term::clear, term::moveTo
