NAME
  expectEQ - assert two values are equal

SYNOPSIS
  expectEQ(actual, expected)

PACKAGE
  testing

IMPORTS
  None. testing assertions are available without an IMPORT statement, but only
  inside a TCASE body.

DESCRIPTION
  expectEQ passes when actual = expected and fails otherwise. It is compiler-
  lowered to the language = operator, so both operands must be comparable with =
  and printable (a scalar, String, Byte, or List OF Byte) so the failure message
  can render them.

  On failure, expectEQ records the detail "expected <expected>, got <actual>",
  aborts the current TCASE, and lets sibling cases and groups continue. The
  aborting error is a reserved internal code the mfb test driver recognizes, so
  the failure is reported as a test failure rather than a crash.

PARAMETERS
  actual
    The value produced by the code under test.

  expected
    The value it should equal. Must be comparable with actual using =.

RETURN VALUE
  Nothing. expectEQ is a statement-level assertion.

ERRORS
  On a failed assertion, raises the reserved internal test-abort error caught by
  the mfb test driver. A genuine runtime error while evaluating actual or expected
  propagates and is reported as a runtime-error failure for the case.

EXAMPLES
  A passing and a failing assertion:

    TESTING
      TGROUP "math"
        TCASE "adds"
          expectEQ(add(2, 3), 5)          ' passes
        END TCASE
        TCASE "reports the mismatch"
          expectEQ(add(2, 3), 6)          ' fails: expected 6, got 5
        END TCASE
      END TGROUP
    END TESTING

SEE ALSO
  expectNQ, expectTrap, expectNTrap
