NAME
  expectEqual - assert two values are equal

SYNOPSIS
  expectEqual(actual, expected)

PACKAGE
  testing

IMPORTS
  None. testing assertions are available without an IMPORT statement, but only
  inside a TCASE body.

DESCRIPTION
  expectEqual passes when actual = expected and fails otherwise. It is the
  generic equality assertion: both operands must be comparable with = and
  printable (a scalar, String, or Byte) so the failure message can render them.
  Integer and Float operands compare numerically. To additionally assert the
  operand type, use the typed forms expectFloat, expectInteger, expectFixed, or
  expectString.

  On failure, expectEqual records the detail "expected <expected>, got <actual>",
  aborts the current TCASE, and lets sibling cases and groups continue. The
  aborting error is a reserved internal code the mfb test driver recognizes, so
  the failure is reported as a test failure rather than a crash.

PARAMETERS
  actual
    The value produced by the code under test.

  expected
    The value it should equal. Must be comparable with actual using =.

RETURN VALUE
  Nothing. expectEqual is a statement-level assertion.

ERRORS
  On a failed assertion, raises the reserved internal test-abort error caught by
  the mfb test driver. A genuine runtime error while evaluating actual or expected
  propagates and is reported as a runtime-error failure for the case.

EXAMPLES
  A passing and a failing assertion:

    TESTING
      TGROUP "math"
        TCASE "adds"
          expectEqual(add(2, 3), 5)          ' passes
        END TCASE
        TCASE "reports the mismatch"
          expectEqual(add(2, 3), 6)          ' fails: expected 6, got 5
        END TCASE
      END TGROUP
    END TESTING

SEE ALSO
  expectNEqual, expectFloat, expectInteger, expectFixed, expectString, expectTrap
