NAME
  expectNQ - assert two values are not equal

SYNOPSIS
  expectNQ(actual, expected)

PACKAGE
  testing

IMPORTS
  None. testing assertions are available without an IMPORT statement, but only
  inside a TCASE body.

DESCRIPTION
  expectNQ passes when actual <> expected and fails otherwise. It is compiler-
  lowered to the language <> operator, so both operands must be comparable with =
  and printable (a scalar, String, Byte, or List OF Byte) so the failure message
  can render them.

  On failure, expectNQ records the detail "expected values to differ, but both
  were <actual>", aborts the current TCASE, and lets sibling cases and groups
  continue.

PARAMETERS
  actual
    The value produced by the code under test.

  expected
    The value it should differ from. Must be comparable with actual using =.

RETURN VALUE
  Nothing. expectNQ is a statement-level assertion.

ERRORS
  On a failed assertion, raises the reserved internal test-abort error caught by
  the mfb test driver. A genuine runtime error while evaluating actual or expected
  propagates and is reported as a runtime-error failure for the case.

EXAMPLES
  Assert a value changed:

    TESTING
      TGROUP "state"
        TCASE "increment changes the value"
          expectNQ(increment(1), 1)
        END TCASE
      END TGROUP
    END TESTING

SEE ALSO
  expectEQ, expectTrap, expectNTrap
