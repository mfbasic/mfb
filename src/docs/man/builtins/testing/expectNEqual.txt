NAME
  expectNEqual - assert two values are not equal

SYNOPSIS
  expectNEqual(actual, expected)

PACKAGE
  testing

IMPORTS
  None. testing assertions are available without an IMPORT statement, but only
  inside a TCASE body.

DESCRIPTION
  expectNEqual passes when actual <> expected and fails otherwise. It is the
  generic inequality assertion: both operands must be comparable with = and
  printable (a scalar, String, or Byte) so the failure message can render them.
  Integer and Float operands compare numerically. To additionally assert the
  operand type, use the typed forms expectNFloat, expectNInteger, expectNFixed,
  or expectNString.

  On failure, expectNEqual records the detail "expected values to differ, but
  both were <actual>", aborts the current TCASE, and lets sibling cases and
  groups continue.

PARAMETERS
  actual
    The value produced by the code under test.

  expected
    The value it should differ from. Must be comparable with actual using =.

RETURN VALUE
  Nothing. expectNEqual is a statement-level assertion.

ERRORS
  On a failed assertion, raises the reserved internal test-abort error caught by
  the mfb test driver. A genuine runtime error while evaluating actual or expected
  propagates and is reported as a runtime-error failure for the case.

EXAMPLES
  Assert a value changed:

    TESTING
      TGROUP "state"
        TCASE "increment changes the value"
          expectNEqual(increment(1), 1)
        END TCASE
      END TGROUP
    END TESTING

SEE ALSO
  expectEqual, expectNFloat, expectNInteger, expectNFixed, expectNString
