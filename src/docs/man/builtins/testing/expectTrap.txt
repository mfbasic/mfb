NAME
  expectTrap - assert that evaluating an expression traps

SYNOPSIS
  expectTrap(expression)
  expectTrap(expression, code)

PACKAGE
  testing

IMPORTS
  None. testing assertions are available without an IMPORT statement, but only
  inside a TCASE body.

DESCRIPTION
  expectTrap evaluates expression inside a trap guard and passes iff the
  evaluation traps. With the two-argument form, it additionally requires the
  trap's error code to equal code (an Integer, typically an errorCode:: constant).

  Because the argument is evaluated under a trap guard, it must be a genuinely-
  fallible call — the same constraint as an inline TRAP. A non-call, an infallible
  call, or an inline-compiled builtin is rejected at compile time; wrap such an
  operation in a FUNC/SUB and pass that call instead.

  On failure (no trap occurred, or the wrong code) expectTrap records the detail,
  aborts the current TCASE, and lets sibling cases and groups continue.

PARAMETERS
  expression
    A fallible call expected to trap. Evaluated for its trap, not its value.

  code (optional)
    An Integer error code the trap must carry. Typically an errorCode:: constant.

RETURN VALUE
  Nothing. expectTrap is a statement-level assertion.

ERRORS
  On a failed assertion, raises the reserved internal test-abort error caught by
  the mfb test driver.

EXAMPLES
  Assert that parsing bad input traps, optionally with a specific code:

    IMPORT errorCode

    TESTING
      TGROUP "parsing"
        TCASE "bad input traps"
          expectTrap(parseInt("nope"))
        END TCASE
        TCASE "bad input traps with a specific code"
          expectTrap(parseInt("nope"), errorCode::ErrInvalidFormat)
        END TCASE
      END TGROUP
    END TESTING
