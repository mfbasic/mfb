NAME
  expectNString - assert two String values are not equal

SYNOPSIS
  expectNString(actual, expected)

PACKAGE
  testing

IMPORTS
  None. testing assertions are available without an IMPORT statement, but only
  inside a TCASE body.

DESCRIPTION
  expectNString passes when both operands are String and actual <> expected; it fails
  otherwise. It is a typed assertion — both actual and expected must be exactly
  String (a mismatched operand type is a compile error), so it asserts the type and
  the value together without any toString. For loose numeric comparison across
  types use expectEqual/expectNEqual instead.

  On failure, expectNString records the mismatch detail, aborts the current TCASE, and lets
  sibling cases and groups continue.

PARAMETERS
  actual
    The String value produced by the code under test.

  expected
    The String value it should not equal.

RETURN VALUE
  Nothing. expectNString is a statement-level assertion.

ERRORS
  On a failed assertion, raises the reserved internal test-abort error caught by
  the mfb test driver.

EXAMPLES
  TESTING
    TGROUP "example"
      TCASE "case"
        expectNString(strings::upper("ab"), "ab")
      END TCASE
    END TGROUP
  END TESTING

SEE ALSO
  expectNEqual, expectString, expectNFloat, expectNInteger, expectNFixed
