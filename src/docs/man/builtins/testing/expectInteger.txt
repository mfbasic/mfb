NAME
  expectInteger - assert two Integer values are equal

SYNOPSIS
  expectInteger(actual, expected)

PACKAGE
  testing

IMPORTS
  None. testing assertions are available without an IMPORT statement, but only
  inside a TCASE body.

DESCRIPTION
  expectInteger passes when both operands are Integer and actual = expected; it fails
  otherwise. It is a typed assertion — both actual and expected must be exactly
  Integer (a mismatched operand type is a compile error), so it asserts the type and
  the value together without any toString. For loose numeric comparison across
  types use expectEqual/expectNEqual instead.

  On failure, expectInteger records the mismatch detail, aborts the current TCASE, and lets
  sibling cases and groups continue.

PARAMETERS
  actual
    The Integer value produced by the code under test.

  expected
    The Integer value it should equal.

RETURN VALUE
  Nothing. expectInteger is a statement-level assertion.

ERRORS
  On a failed assertion, raises the reserved internal test-abort error caught by
  the mfb test driver.

EXAMPLES
  TESTING
    TGROUP "example"
      TCASE "case"
        expectInteger(math::abs(0 - 5), 5)
      END TCASE
    END TGROUP
  END TESTING

SEE ALSO
  expectEqual, expectNInteger, expectFloat, expectFixed, expectString
