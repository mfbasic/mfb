NAME
  expectNFloat - assert two Float values are not equal

SYNOPSIS
  expectNFloat(actual, expected)

PACKAGE
  testing

IMPORTS
  None. testing assertions are available without an IMPORT statement, but only
  inside a TCASE body.

DESCRIPTION
  expectNFloat passes when both operands are Float and actual <> expected; it fails
  otherwise. It is a typed assertion — both actual and expected must be exactly
  Float (a mismatched operand type is a compile error), so it asserts the type and
  the value together without any toString. For loose numeric comparison across
  types use expectEqual/expectNEqual instead.

  On failure, expectNFloat records the mismatch detail, aborts the current TCASE, and lets
  sibling cases and groups continue.

PARAMETERS
  actual
    The Float value produced by the code under test.

  expected
    The Float value it should not equal.

RETURN VALUE
  Nothing. expectNFloat is a statement-level assertion.

ERRORS
  On a failed assertion, raises the reserved internal test-abort error caught by
  the mfb test driver.

EXAMPLES
  TESTING
    TGROUP "example"
      TCASE "case"
        expectNFloat(math::sqrt(2.0), 2.0)
      END TCASE
    END TGROUP
  END TESTING

SEE ALSO
  expectNEqual, expectFloat, expectNInteger, expectNFixed, expectNString
