NAME
  regex - match, search, and replace text with regular expressions

SYNOPSIS
  IMPORT regex
  regex::match(value, pattern)
  regex::find(value, pattern, start)
  regex::findAll(value, pattern, start)
  regex::replace(value, pattern, replacement)

DESCRIPTION
  The regex package searches and rewrites text with a single, portable
  regular-expression dialect that is MFBASIC's own: its syntax and semantics are
  defined entirely by `./mfb spec stdlib regex` and produce byte-for-byte identical
  results on every target, never deferring to a host libc, locale, or OS regex
  library. regex is a built-in package: IMPORT regex needs no manifest
  dependency. pattern and replacement are ordinary runtime String values, so they
  may be built or read at run time; an invalid pattern fails with ErrInvalidFormat
  rather than being silently accepted.

  Matching operates over Unicode scalar values. Every position and index a regex
  function reports or accepts is a zero-based Unicode scalar index, never a byte
  offset and never a grapheme-cluster index, consistent with len and the strings
  package. A string of n scalars has positions 0 .. n; position n is after the
  last scalar. The four functions differ only in what they report: match returns
  a Boolean (does the pattern match anywhere), find returns the start index of the
  first match at or after start, findAll returns a List OF Integer of the start
  index of every non-overlapping match, and replace returns a new String with
  every non-overlapping match rewritten.

  PATTERN LANGUAGE. A pattern is built from these pieces:

    Literals          Any scalar that is not a metacharacter matches itself. The
                      metacharacters outside a class are \ . * + ? ( ) [ ] { } ^
                      $ |; write one literally by escaping it (\. \* \\) .

    Concatenation     Writing atoms in sequence matches them left to right: abc.

    Alternation       a|b|c matches the first alternative that lets the whole
                      pattern match (preference order, left to right).

    Grouping          (R) is a capturing group, numbered 1, 2, ... by opening
                      parenthesis (group 0 is the whole match). (?:R) groups
                      without capturing. (?<name>R) and (?P<name>R) are named
                      capturing groups, usable in replacement (see below).

    Quantifiers       X* X+ X? repeat the preceding atom (zero-or-more, one-or-
                      more, zero-or-one). X{m}, X{m,}, X{m,n} are counted forms
                      (m,n non-negative, m <= n). Quantifiers are greedy by
                      default; a trailing ? makes them lazy (X*?, X+?, X{m,n}?).
                      A quantifier with no preceding atom, or two stacked on one
                      atom, is invalid.

    Dot               . matches any one scalar except newline (U+000A); under the
                      s flag it matches any scalar including newline.

    Character class   [abc] matches one listed scalar; [a-z] a scalar in an
                      inclusive range; [^...] any scalar NOT listed. Inside a
                      class the active metacharacters are ] \ ^ (negation, first
                      only) and - (range, between endpoints). Classes may contain
                      shorthands and properties ([\d.], [\p{L}_]) and POSIX names
                      ([[:alpha:]_], [[:^digit:]]). An empty class [] or [^] is
                      invalid.

    Class shorthands  \d (decimal digit), \w (word scalar: letter, mark, digit,
                      connector punctuation, or joiner), \s (whitespace), and the
                      negations \D \W \S. Their member sets are the pinned Unicode
                      definitions, identical on every target.

    Anchors           ^ and $ match start and end of input; under the m flag they
                      also match at line boundaries. \A and \z always match the
                      absolute start and end. \b and \B match a word boundary and
                      its negation. All anchors are zero-width.

    Escapes           Beyond the class shorthands and anchors: \n \r \t \f \v \a
                      \e \0 for control scalars; \xHH and \x{H..H} for a scalar by
                      hex value; \p{Prop}/\P{Prop} (and \pX/\PX) for Unicode
                      general categories, scripts, and binary properties; and a
                      backslash before ASCII punctuation for that punctuation
                      literally. A backslash before any other ASCII letter or
                      digit is invalid -- there is no "unknown escape is literal"
                      fallback.

    Flags             i (case-insensitive, Unicode simple case folding), m
                      (multiline ^/$), s (dot matches newline), U (swap greedy and
                      lazy), x (verbose: ignore unescaped whitespace and # line
                      comments). Set them with a directive (?flags) for the rest
                      of the enclosing group, or scope them to one group with
                      (?flags:R). Turn flags off after a dash: (?-i), (?i-s:R).

  MATCH SELECTION. For any search there is at most one reported match: the one
  beginning at the smallest position where any match exists (leftmost), and at
  that position the one chosen by preference order -- earlier alternatives,
  greedy quantifiers taking as much as possible, and lazy quantifiers taking as
  little as possible, all subject to the overall match succeeding. find reports
  that match's start; findAll and replace iterate it left to right over non-
  overlapping matches. A zero-length match is valid; the iterator advances one
  scalar past an empty match so iteration always terminates and never reports two
  empty matches at the same position.

  REPLACEMENT. The replacement argument to replace is literal text interleaved
  with capture references: $N or ${N} inserts capturing group N ($0 the whole
  match), $name or ${name} inserts a named group, and $$ inserts a literal $. An
  unbraced reference consumes the longest valid run, so use the braced form to
  butt a reference against following text (${1}0 is group 1 then "0", whereas $10
  is group 10). A reference to a group that did not participate, or to an unknown
  name or out-of-range number, expands to the empty string, so replace never
  fails on replacement content; only an invalid pattern fails.

  UNICODE. All Unicode-dependent behavior -- the shorthand classes, \p{...}
  properties, case folding, and so on -- is resolved against a single Unicode
  version pinned by the implementation and identical across every target and code
  path. The host's Unicode tables are never consulted.

  NON-GOALS. Backreferences (\1, \k<name>) and look-around (lookahead (?=...),
  (?!...); lookbehind (?<=...), (?<!...)) are not supported and are rejected as
  invalid patterns -- they are not parsed as assertions. Note that (?<name>...)
  is a valid named group, distinct from the lookbehind heads (?<=/(?<!, which
  differ only in whether < is followed by a name or by = / !.

ERRORS
  77050003 (ErrInvalidFormat)
  Raised by every regex function when pattern is not a valid pattern: an
  unbalanced or unterminated group or class, a quantifier with no atom or stacked
  quantifiers, a counted quantifier with m > n, a class range whose low endpoint
  exceeds its high endpoint, an empty class, a backslash escape outside the
  defined set, a \x.../\x{...} value that is not a valid scalar, an unknown
  \p{...} property, a malformed flag or group head, or a non-goal construct
  (backreference or look-around).

  77050001 (ErrIndexOutOfRange)
  Raised by find and findAll when start is outside 0 .. len(value). Pattern
  compilation failure (ErrInvalidFormat) takes precedence when both apply.

  No regex function fails on the absence of a match: match returns FALSE, find
  returns -1, findAll returns an empty list, and replace returns value unchanged.
  ErrNotFound is never raised by this package.
