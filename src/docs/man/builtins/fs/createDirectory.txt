NAME
  createDirectory - create a single directory

SYNOPSIS
  fs::createDirectory(path AS String) AS Nothing

PACKAGE
  fs

IMPORTS
  IMPORT fs

DESCRIPTION
  Creates the single directory named by path with one host mkdir operation. On
  success the directory exists and the function returns Nothing.

  Only the final component is created; the parent directory must already exist.
  createDirectory does not create intermediate directories, so a path whose
  parent is missing fails rather than building the chain. Use
  fs::createDirectories to create a directory along with any missing parents,
  like "mkdir -p".

  The new directory is requested with permission bits 0755 (rwxr-xr-x), which
  the host masks with the process umask in the usual way, so the directory's
  actual mode is 0755 with the umask bits cleared.

  path is interpreted as UTF-8 bytes and passed to the host filesystem. It may
  be absolute or relative to the current working directory and may contain
  Unicode characters, including emoji, when the host filesystem accepts those
  names. The string must not be empty and must not contain an embedded NUL byte,
  because the host call requires a NUL-terminated path; an internal
  NUL-terminated copy of the path is allocated for the call.

  createDirectory never overwrites or reuses an existing entry: if anything
  already exists at path, including an existing directory, the call fails with
  ErrAlreadyExists rather than succeeding quietly. When the host refuses the
  operation for any reason the failure is mapped to the matching error below and
  the filesystem is left unchanged.

PARAMETERS
  path AS String
    The filesystem path of the directory to create. Interpreted as UTF-8 bytes;
    may be absolute or relative to the current working directory. Must be
    non-empty and free of embedded NUL bytes. Only the final component is
    created; every parent component must already exist as a directory.

RETURN VALUE
  AS Nothing
    Returns nothing on success, after the directory named by path has been
    created.

ERRORS
  77050002 (ErrInvalidArgument)
  Raised when path is empty or contains an embedded NUL byte, so it cannot be
  turned into a valid NUL-terminated host path.

  77010001 (ErrOutOfMemory)
  Raised when the internal NUL-terminated copy of path cannot be allocated.

  77030001 (ErrPathNotFound)
  Raised when a parent component of path does not exist (host ENOENT).

  77030003 (ErrAccessDenied)
  Raised when the host denies permission to create the directory (host EACCES).

  77050005 (ErrAlreadyExists)
  Raised when an entry already exists at path, including an existing directory
  (host EEXIST).

  77030005 (ErrDirectoryNotEmpty)
  Raised when the host reports a non-empty directory conflict for the operation
  (host ENOTEMPTY).

  77030002 (ErrInvalidPath)
  Raised when path is unusable as a path, including a non-directory used as a
  directory component, an over-long path, an invalid byte sequence, or a symlink
  loop (host ENOTDIR, ENAMETOOLONG, EILSEQ, or ELOOP).

  77020002 (ErrOutput)
  Raised when the host refuses the operation for any other reason.

EXAMPLES
  Create a single output directory whose parent already exists:

    fs::createDirectory("target/example")

  Guard against re-creating a directory that already exists:

    IF NOT fs::directoryExists("target/cache") THEN
      fs::createDirectory("target/cache")
    END IF

SEE ALSO
  fs::createDirectories, fs::deleteDirectory, fs::directoryExists, fs::exists
