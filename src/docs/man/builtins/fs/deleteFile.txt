NAME
  deleteFile - remove a file from the filesystem

SYNOPSIS
  fs::deleteFile(path AS String) AS Nothing

PACKAGE
  fs

IMPORTS
  IMPORT fs

DESCRIPTION
  Removes the filesystem entry named by path with a single host unlink
  operation. On success the entry is gone and the function returns Nothing.

  When the final component of path is a symbolic link, the link itself is
  removed rather than the file it points to. The function only removes a single
  non-directory entry; it does not recurse and it does not remove directories.
  Use fs::deleteDirectory to remove a directory.

  path is interpreted as UTF-8 bytes and passed to the host filesystem. It may
  be absolute or relative to the current working directory and may contain
  Unicode characters when the host filesystem accepts those names. The string
  must not be empty and must not contain an embedded NUL byte, because the host
  call requires a NUL-terminated path; an internal NUL-terminated copy of the
  path is allocated for the call.

  Attempting to remove an entry that is a directory is reported as a host
  failure rather than as a directory-specific error, since unlink does not
  operate on directories. When the host refuses the removal for any reason the
  failure is mapped to the matching error below and path is left unchanged.

PARAMETERS
  path AS String
    The filesystem path of the entry to remove. Interpreted as UTF-8 bytes; may
    be absolute or relative to the current working directory. Must be non-empty
    and free of embedded NUL bytes. A trailing symlink is removed rather than
    followed.

RETURN VALUE
  AS Nothing
    Returns nothing on success, after the entry named by path has been removed.

ERRORS
  77050002 (ErrInvalidArgument)
  Raised when path is empty or contains an embedded NUL byte, so it cannot be
  turned into a valid NUL-terminated host path.

  77010001 (ErrOutOfMemory)
  Raised when the internal NUL-terminated copy of path cannot be allocated.

  77030001 (ErrPathNotFound)
  Raised when no entry exists at path (host ENOENT).

  77030003 (ErrAccessDenied)
  Raised when the host denies permission to remove the entry (host EACCES).

  77030002 (ErrInvalidPath)
  Raised when path is unusable as a path, including a non-directory used as a
  directory component, an over-long path, an invalid byte sequence, or a symlink
  loop (host ENOTDIR, ENAMETOOLONG, EILSEQ, or ELOOP).

  77050005 (ErrAlreadyExists)
  Raised when the host reports that the target already exists in a conflicting
  form (host EEXIST).

  77030005 (ErrDirectoryNotEmpty)
  Raised when the host reports the entry is a non-empty directory (host
  ENOTEMPTY).

  77020002 (ErrOutput)
  Raised when the host refuses the removal for any other reason, such as path
  naming a directory or an otherwise non-removable entry.

EXAMPLES
  Remove a generated output file:

    fs::deleteFile("target/output.txt")

  Write a file and then remove it:

    fs::writeText("scratch.txt", "temporary")
    fs::deleteFile("scratch.txt")
