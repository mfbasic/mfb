NAME
  flush - drain a File's output buffer to disk

SYNOPSIS
  fs::flush(file AS File) AS Nothing

PACKAGE
  fs

IMPORTS
  IMPORT fs

DESCRIPTION
  Drains any output buffered for file, writing the held bytes to the underlying
  file descriptor at once. This matters only when the handle has buffering enabled
  with fs::setBuffered(file, TRUE); on an unbuffered handle (the default) there is
  nothing held back, so fs::flush is a no-op.

  Use fs::flush at a checkpoint where buffered data must reach the file before the
  program continues — for example before another process reads the file, or before
  a long pause. Closing the handle (via fs::close or scope exit) also flushes, so an
  explicit flush is only needed mid-stream; the final bytes are never lost to a
  clean close.

  Buffering and flushing are per handle: fs::flush(file) drains only file's buffer
  and does not affect any other open File.

PARAMETERS
  file AS File
    An open File resource whose output buffer should be drained.

RETURN VALUE
  AS Nothing
    Returns nothing. The function is invoked for its side effect of draining the
    handle's output buffer.

ERRORS
  77020002 (ErrOutput)
  Raised when the underlying write of the buffered bytes fails, for example when the
  disk is full or the descriptor is no longer writable.

EXAMPLES
  Force buffered data to disk at a checkpoint, then keep writing:

    fs::setBuffered(file, TRUE)
    fs::writeAll(file, header)
    fs::flush(file)
    fs::writeAll(file, body)

SEE ALSO
  fs::setBuffered, fs::isBuffered, fs::writeAll, fs::close
