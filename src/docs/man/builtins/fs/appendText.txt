NAME
  appendText - append a String to the end of a file as UTF-8 text

SYNOPSIS
  fs::appendText(path AS String, value AS String) AS Nothing

PACKAGE
  fs

IMPORTS
  IMPORT fs

DESCRIPTION
  Opens the file named by path for appending, creating it with no contents if it
  does not already exist, writes the complete contents of value as UTF-8 text
  after whatever the file already held, flushes the file to disk, closes it, and
  returns nothing. Any existing contents are preserved and the new text is added
  after them; to replace a file's contents instead of extending them, use
  fs::writeText.

  The file is opened in append mode, so every write is positioned at the current
  end of the file. An empty String leaves the file unchanged in length, creating
  it as an empty file if it did not exist. A String already holds well-formed
  UTF-8, so the bytes are written exactly as held, with no re-encoding, decoding,
  or newline translation, and no trailing newline is added.

  The text payload is written directly from the String's packed byte data. The
  write is retried until every byte has been written or the host reports an
  output failure, so a short host write that transfers only part of the buffer is
  resumed rather than treated as complete.

  When the file is created it is given mode 438 (octal 0666) before the process
  umask is applied; an existing file keeps its current mode. The file is created
  and opened only after path has been validated, and the final path component is
  followed when it is a symlink, so appending through a symlink appends to the
  target file.

  path is interpreted as UTF-8 bytes and passed to the host filesystem. It may be
  absolute or relative to the current working directory and may contain Unicode
  characters when the host filesystem accepts those names. The string must not be
  empty and must not contain an embedded NUL byte, because the host open call
  requires a NUL-terminated path.

  The file is closed before the function returns on both the success and the
  write-failure paths. The append is not atomic: a reader observing the file
  while the write is in progress may see only part of the appended text, and a
  failure partway through leaves the file extended by only the bytes written so
  far.

PARAMETERS
  path AS String
    The filesystem path of the file to append to. Interpreted as UTF-8 bytes; may
    be absolute or relative to the current working directory. Must be non-empty
    and free of embedded NUL bytes. The file is created if it does not exist.

  value AS String
    The text to append. Written verbatim as the String's UTF-8 bytes, in order,
    after the file's existing contents. An empty String leaves the file's length
    unchanged.

RETURN VALUE
  AS Nothing
    Returns nothing on success, after every byte has been written, flushed, and
    the file has been closed.

ERRORS
  77050002 (ErrInvalidArgument)
  Raised when path is empty or contains an embedded NUL byte, so it cannot be
  turned into a valid NUL-terminated host path.

  77010001 (ErrOutOfMemory)
  Raised when the internal NUL-terminated copy of path cannot be allocated.

  77050004 (ErrNotFound)
  Raised when the file cannot be opened because a component of the path does not
  exist, such as a missing parent directory (host ENOENT).

  77030003 (ErrAccessDenied)
  Raised when the host denies permission to create or open the file (host EACCES).

  77020002 (ErrOutput)
  Raised when the file cannot be opened for any other host reason, including path
  refers to a directory or another non-writable target, and when writing,
  flushing, or closing the file fails partway through.

EXAMPLES
  Append a line to a log file:

    fs::appendText("target/output.txt", "line\n")

  Build up a file across several calls:

    fs::appendText("notes.txt", "first\n")
    fs::appendText("notes.txt", "second\n")
    LET text AS String = fs::readText("notes.txt")
    io::print(text)
