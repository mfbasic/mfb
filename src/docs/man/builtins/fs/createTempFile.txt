NAME
  createTempFile - securely create and open a unique temporary file

SYNOPSIS
  fs::createTempFile() AS File
  fs::createTempFile(directory AS String) AS File

PACKAGE
  fs

IMPORTS
  IMPORT fs

DESCRIPTION
  Creates a brand-new file with a unique, unpredictable name and returns an open
  File resource referring to it. The file name has the form mfb-<uuid>.tmp, where
  <uuid> is a freshly generated, randomly seeded version 4 UUID rendered in the
  canonical 8-4-4-4-12 hexadecimal form. Because the name is drawn from operating
  system randomness, two calls effectively never collide and the name cannot be
  guessed by another process.

  Without an argument the file is created inside the host's temporary directory,
  the same location returned by fs::tempDirectory. With a directory argument the
  file is created directly inside that directory; the argument names the
  containing directory, not the file, and no name component of your own is added.
  The directory must already exist and be writable, since the new file is created
  there. The argument is interpreted as UTF-8 bytes, may be absolute or relative
  to the current working directory, and may contain Unicode characters when the
  host filesystem accepts them. It must not be empty and must not contain an
  embedded NUL byte, because the host open call requires a NUL-terminated path.

  The file is opened read/write with exclusive-create semantics and permission
  bits 0600, so the call always yields a freshly created, empty file readable and
  writable only by the current user. Exclusive creation means the call fails
  rather than reusing or truncating any pre-existing file, which together with
  the random name closes the classic temporary-file race and symlink-redirection
  attacks. The random name is generated from host entropy before the file is
  opened; a failure to obtain those random bytes prevents the file from being
  created.

  The returned File is positioned at the start of the empty file and is owned by
  the caller. It is closed by lexical drop when the binding that holds it leaves
  scope, or explicitly with fs::close. The file itself is not deleted on close;
  removing it is the caller's responsibility, for example with fs::deleteFile.

OVERLOADS
  fs::createTempFile() AS File
    Creates the temporary file in the host temporary directory reported by
    fs::tempDirectory.

  fs::createTempFile(directory AS String) AS File
    Creates the temporary file inside the given directory instead of the host
    temporary directory.

PARAMETERS
  directory AS String
    The path of the directory in which to create the temporary file. Interpreted
    as UTF-8 bytes; may be absolute or relative to the current working directory.
    Must be non-empty, free of embedded NUL bytes, and must name an existing,
    writable directory. When omitted, the host temporary directory is used.

RETURN VALUE
  AS File
    An open, empty File resource for a newly created file opened read/write with
    0600 permissions and positioned at the start of the file. The resource must
    eventually be closed, by scope drop or by fs::close.

ERRORS
  77050002 (ErrInvalidArgument)
  Raised when the directory argument is empty or contains an embedded NUL byte,
  so it cannot be turned into a valid NUL-terminated host path.

  77010001 (ErrOutOfMemory)
  Raised when the internal buffer for the constructed temporary path or the File
  resource record cannot be allocated.

  77050004 (ErrNotFound)
  Raised when the file cannot be created because a path component does not exist,
  such as a missing containing directory (host ENOENT).

  77030003 (ErrAccessDenied)
  Raised when the host denies permission to create the file in the directory
  (host EACCES).

  77050005 (ErrAlreadyExists)
  Raised when the exclusive create fails because a file with the generated name
  already exists (host EEXIST).

  77020002 (ErrOutput)
  Raised when random bytes for the name cannot be obtained, and when the host
  refuses to create the file for any other reason not classified above.

EXAMPLES
  Create a temporary file in the host temporary directory and write to it:

    RES f = fs::createTempFile()
    fs::writeAll(f, "data")
    ' f is closed by lexical drop when this scope ends

  Create a temporary file in a specific directory:

    RES g = fs::createTempFile("target")
    fs::writeAll(g, "data")
    ' g is closed by lexical drop when this scope ends
