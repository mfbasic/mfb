NAME
  setBuffered - enable or disable output buffering for a File

SYNOPSIS
  fs::setBuffered(file AS File, enabled AS Boolean) AS Nothing

PACKAGE
  fs

IMPORTS
  IMPORT fs

DESCRIPTION
  Turns opt-in output buffering on or off for a single open File. Buffering is off
  by default: every fs::writeAll and fs::writeAllBytes reaches the operating system
  immediately, exactly as without this call. After fs::setBuffered(file, TRUE),
  incremental writes to file are instead held in a per-handle buffer and written in
  larger blocks, collapsing a loop of small writes into roughly one write per full
  buffer.

  When buffering is on, held output is drained automatically when the buffer fills,
  on fs::flush(file), and — most importantly — when the handle is closed, whether by
  fs::close or by lexical scope exit of its RES binding. Passing FALSE drains any
  pending bytes first and then disables buffering, so switching buffering off never
  strands data.

  Buffering is per handle: each open File has its own buffer and enabled flag, and
  the setting affects only file. Only incremental fs::writeAll / fs::writeAllBytes
  writes are buffered; whole-file writes (fs::writeText, fs::writeBytes, the append
  and atomic variants) already issue a single write and ignore this setting.

  Because buffered output is held in memory until drained, a hard crash (SIGSEGV,
  SIGKILL, or an abort) can lose bytes that were written but not yet flushed. Close
  or flush a buffered handle to make its data durable; leave buffering off (the
  default) when partial-output-on-crash durability matters. A buffered handle should
  also be flushed before it is transferred to another thread, which resets it to
  unbuffered.

PARAMETERS
  file AS File
    An open File resource to configure.
  enabled AS Boolean
    TRUE to enable output buffering for this handle, FALSE to drain any pending
    output and disable it.

RETURN VALUE
  AS Nothing
    Returns nothing. The function is invoked for its side effect of changing the
    handle's buffering mode (and, when disabling, draining any pending output).

ERRORS
  No errors. A write failure while draining on the disable transition is not
  reported here; it surfaces from the next fs::flush, buffered write, or close.

EXAMPLES
  Buffer a loop of small writes and let scope exit flush and close:

    RES log = fs::openFile("events.log", "write")
    fs::setBuffered(log, TRUE)
    FOR EACH event IN events
      fs::writeAll(log, event & "\n")
    NEXT
    ' log is flushed and closed automatically at scope exit
