NAME
  setCurrentDirectory - change the process current working directory

SYNOPSIS
  fs::setCurrentDirectory(path AS String) AS Nothing

PACKAGE
  fs

IMPORTS
  IMPORT fs

DESCRIPTION
  Changes the current working directory of the running process to the directory
  named by path with one host chdir operation. On success the working directory
  has been changed and the function returns Nothing.

  The change affects the whole process, so every relative path passed to later
  fs:: functions, including fs::open, fs::readText, fs::canonicalPath, and
  fs::listDirectory, resolves against the new directory rather than the old one.
  The new value can be read back with fs::currentDirectory.

  path is interpreted as UTF-8 bytes and passed to the host filesystem. It may be
  absolute or relative to the current working directory; a relative path, such as
  "tests" or "..", is resolved against the existing working directory before the
  change takes effect. The path may contain Unicode characters, including emoji,
  when the host filesystem accepts those names. The string must not be empty and
  must not contain an embedded NUL byte, because the host call requires a
  NUL-terminated path; an internal NUL-terminated copy of the path is allocated
  for the call.

  The named entry must exist and must be a directory the process is allowed to
  enter; every component leading to it must itself be a traversable directory.
  When the host refuses the operation for any reason the failure is mapped to the
  matching error below and the working directory is left unchanged.

PARAMETERS
  path AS String
    The filesystem path of the directory to become the new working directory.
    Interpreted as UTF-8 bytes; may be absolute or relative to the current
    working directory. Must be non-empty and free of embedded NUL bytes. The
    entry must exist and be a directory the process can enter.

RETURN VALUE
  AS Nothing
    Returns nothing on success, after the process working directory has been
    changed to the directory named by path.

ERRORS
  77050002 (ErrInvalidArgument)
  Raised when path is empty or contains an embedded NUL byte, so it cannot be
  turned into a valid NUL-terminated host path.

  77010001 (ErrOutOfMemory)
  Raised when the internal NUL-terminated copy of path cannot be allocated.

  77030001 (ErrPathNotFound)
  Raised when path, or a component of it, does not exist (host ENOENT).

  77030003 (ErrAccessDenied)
  Raised when the host denies permission to enter path or a component leading to
  it (host EACCES).

  77050005 (ErrAlreadyExists)
  Raised when the host reports an existing-entry conflict for the operation (host
  EEXIST).

  77030005 (ErrDirectoryNotEmpty)
  Raised when the host reports a non-empty directory conflict for the operation
  (host ENOTEMPTY).

  77030002 (ErrInvalidPath)
  Raised when path is unusable as a directory, including a non-directory used as
  a directory component, an over-long path, an invalid byte sequence, or a
  symlink loop (host ENOTDIR, ENAMETOOLONG, EILSEQ, or ELOOP).

  77020002 (ErrOutput)
  Raised when the host refuses the operation for any other reason.

EXAMPLES
  Move into a subdirectory and back up to the parent:

    fs::setCurrentDirectory("tests")
    fs::setCurrentDirectory("..")

  Confirm the move by reading back the working directory:

    fs::setCurrentDirectory("target")
    LET here AS String = fs::currentDirectory()
    io::print(here)

SEE ALSO
  fs::currentDirectory, fs::canonicalPath, fs::directoryExists, fs::tempDirectory
