NAME
  close - close an open File resource and release its OS handle

SYNOPSIS
  fs::close(file AS File) AS Nothing

PACKAGE
  fs

IMPORTS
  IMPORT fs

DESCRIPTION
  Closes file, first draining any MFBASIC output buffer attached to the handle
  (see fs::setBuffered) so buffered on-disk data is never stranded, then flushing
  any data buffered by the host and releasing the underlying operating-system file
  descriptor. The File is marked closed before the function returns, so any later
  fs:: call that takes the same File raises an error rather than touching a stale
  descriptor. A File is also closed automatically when its RES binding leaves
  scope, which drains the buffer the same way.

  Closing is otherwise automatic. Every File returned by fs::open, fs::openFile,
  fs::openFileNoFollow, or fs::createTempFile is closed by lexical drop when the
  binding that holds it leaves scope. Call fs::close only when the descriptor
  must be released earlier than that, for example to reopen the same path, to
  observe writes from another process, or to bound the number of descriptors a
  long-running program holds open at once. Closing a File and then letting it
  drop is safe: the drop sees the closed flag and does nothing.

  The function reads and writes no file contents of its own beyond the flush the
  host performs as part of releasing the descriptor. It is an error to close a
  File that is already closed, including one closed by a previous fs::close call
  on the same value.

PARAMETERS
  file AS File
    The open File resource to close, as returned by fs::open, fs::openFile,
    fs::openFileNoFollow, or fs::createTempFile. Must not already be closed.

RETURN VALUE
  AS Nothing
    Returns nothing on success. After a successful return the descriptor is
    released and file is marked closed and must not be used again.

ERRORS
  77030004 (ErrResourceClosed)
  Raised when file has already been closed, whether by an earlier fs::close call
  on the same value or by a prior scope-drop.

  77030006 (ErrCloseFailed)
  Raised when the host OS reports a failure while flushing or releasing the
  descriptor.

EXAMPLES
  Open a file and release its handle explicitly:

    LET f AS File = fs::openFile("data.txt")
    LET line AS String = fs::readLine(f)
    fs::close(f)

  Write a file, then close it before reopening the same path:

    LET w AS File = fs::open("out.txt", "write")
    fs::writeAll(w, "hello")
    fs::close(w)
    LET r AS File = fs::open("out.txt", "read")
    io::print(fs::readAll(r))
    fs::close(r)
