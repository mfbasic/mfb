NAME
  writeText - write a String to a file as UTF-8 text, replacing its contents

SYNOPSIS
  fs::writeText(path AS String, value AS String) AS Nothing

PACKAGE
  fs

IMPORTS
  IMPORT fs

DESCRIPTION
  Opens the file named by path for writing, truncating it to empty if it already
  exists or creating it if it does not, writes the complete contents of value as
  UTF-8 text, flushes the file to disk, closes it, and returns nothing. Any
  previous contents of an existing file are discarded; to add to a file instead
  of replacing it, use fs::appendText.

  The text payload is written directly from the String's packed byte data. A
  String already holds well-formed UTF-8, so the bytes are written exactly as
  held, with no re-encoding, decoding, or newline translation. The write is
  retried until every byte has been written or the host reports an output
  failure, so a short host write that transfers only part of the buffer is
  resumed rather than treated as complete. An empty String produces an empty
  (truncated) file.

  The new file is created with mode 438 (octal 0666) before the process umask is
  applied. The file is created and truncated only after path has been validated,
  and the final path component is followed when it is a symlink, so writing
  through a symlink writes the target file.

  path is interpreted as UTF-8 bytes and passed to the host filesystem. It may be
  absolute or relative to the current working directory and may contain Unicode
  characters when the host filesystem accepts those names. The string must not be
  empty and must not contain an embedded NUL byte, because the host open call
  requires a NUL-terminated path.

  The file is closed before the function returns on both the success and the
  write-failure paths. The write is not atomic: a reader observing the file while
  the write is in progress may see a partially written file, and a failure
  partway through leaves the file truncated and partially written. For an
  all-or-nothing replacement, use fs::writeTextAtomic.

PARAMETERS
  path AS String
    The filesystem path of the file to write. Interpreted as UTF-8 bytes; may be
    absolute or relative to the current working directory. Must be non-empty and
    free of embedded NUL bytes.

  value AS String
    The text to write. Written verbatim as the String's UTF-8 bytes, in order.
    An empty String truncates the file to zero length.

RETURN VALUE
  AS Nothing
    Returns nothing on success, after every byte has been written, flushed, and
    the file has been closed.

ERRORS
  77050002 (ErrInvalidArgument)
  Raised when path is empty or contains an embedded NUL byte, so it cannot be
  turned into a valid NUL-terminated host path.

  77010001 (ErrOutOfMemory)
  Raised when the internal NUL-terminated copy of path cannot be allocated.

  77050004 (ErrNotFound)
  Raised when the file cannot be created because a component of the path does not
  exist, such as a missing parent directory (host ENOENT).

  77030003 (ErrAccessDenied)
  Raised when the host denies permission to create or open the file (host EACCES).

  77050005 (ErrAlreadyExists)
  Raised when the file cannot be opened because the target already exists in a
  form that conflicts with creating it (host EEXIST).

  77020002 (ErrOutput)
  Raised when the file cannot be opened for any other host reason, including path
  refers to a directory or another non-writable target, and when writing,
  flushing, or closing the file fails partway through.

EXAMPLES
  Write text to a file:

    fs::writeText("target/output.txt", "Hello")

  Replace a file's contents and read them back:

    fs::writeText("greeting.txt", "hello")
    LET text AS String = fs::readText("greeting.txt")
    io::print(text)
