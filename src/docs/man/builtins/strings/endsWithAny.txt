NAME
  endsWithAny - test whether a string ends with any of several suffixes

SYNOPSIS
  strings::endsWithAny(value AS String, suffixes AS List OF String) AS Boolean

PACKAGE
  strings

IMPORTS
  IMPORT strings

DESCRIPTION
  Returns TRUE when value ends with at least one of the strings in suffixes, and
  FALSE otherwise. The candidate suffixes are tested in list order and the
  function stops at the first one that matches; the particular suffix that
  matched is not reported, only whether some suffix did.

  Each individual test is the same raw byte comparison performed by
  strings::endsWith: the trailing bytes of value are compared against the bytes
  of the candidate suffix, in order, and that candidate matches only when every
  one of its bytes equals the byte at the same offset from the end of value.
  Because both operands are Strings holding UTF-8 bytes, a matching byte suffix
  that is itself a well-formed UTF-8 string corresponds to a suffix at character
  boundaries; the comparison is performed on bytes and does not normalize,
  case-fold, or otherwise transform either operand. A candidate longer than
  value cannot match and is skipped. The empty string as a candidate always
  matches, so a suffixes list that contains "" makes the result TRUE for any
  value, including the empty string.

  When suffixes is empty there are no candidates to match and the result is
  FALSE. Neither value nor the list is modified and the function has no side
  effects. This function never fails. To test a single suffix use
  strings::endsWith; to test leading bytes against several candidates use
  strings::startsWithAny.

PARAMETERS
  value AS String
    The string whose trailing bytes are examined. May be empty.

  suffixes AS List OF String
    The candidate suffixes to look for at the end of value, tested in order.
    May be empty, in which case the function returns FALSE. Individual entries
    may be empty; an empty entry always matches.

RETURN VALUE
  AS Boolean
    TRUE when the bytes of some entry in suffixes match the trailing bytes of
    value, FALSE otherwise. An empty suffixes list returns FALSE; a list
    containing the empty string always returns TRUE.

ERRORS
  No errors.

EXAMPLES
  Test a filename against several image extensions:

    IMPORT io
    IMPORT strings

    FUNC main AS Integer
      LET suffixes = [".png", ".jpg", ".gif"]
      io::print(toString(strings::endsWithAny("photo.png", suffixes)))
      RETURN 0
    END FUNC

  An empty list never matches:

    LET hit AS Boolean = strings::endsWithAny("anything", [])

SEE ALSO
