# IR Lowering

The typed, architecture-independent IR: the operation and value forms shared by both back ends.

IR lowering is implemented in `src/ir.rs`.

The IR is a typed, architecture-independent representation of the concrete AST.
It contains:

- Project name.
- Optional executable entry point.
- Top-level bindings (program globals).
- User-defined types.
- Functions.
- Parameters and defaults.
- Structured operations.
- Structured expression values.
- Native resources (`native_resources`), native `LINK` functions
  (`link_functions`), and link aliases (`link_aliases`) for the native back end.
- Documentation (`docs`) extracted from `DOC` blocks.

The main IR operation forms are:[[src/ir/op.rs:IrOp]]

- `Bind`
- `Assign`
- `AssignGlobal` — assignment to a top-level binding
- `StateAssign` — replace the `STATE` payload of a `RES` binding
- `Return`
- `ExitLoop` — structured `EXIT FOR/DO/WHILE` (carries the `LoopKind`)
- `ContinueLoop` — structured `CONTINUE FOR/DO/WHILE` (carries the `LoopKind`)
- `ExitProgram` — `EXIT PROGRAM` with a status code and full RAII unwind
- `Fail`
- `Eval`
- `If`
- `Match`
- `While` — `WHILE`/`DO WHILE` loops (carries the `LoopKind`)
- `For` — counted `FOR` loops with start/end/step
- `DoUntil` — bottom-tested `DO ... UNTIL` loops
- `ForEach`
- `Trap` — inline `TRAP` block; `RECOVER` is lowered into the trap body via
  recover targets rather than a dedicated op

The main IR value forms are:[[src/ir/value.rs:IrValue]]

- `Const` — typed literal constants
- `Local` — local variable references
- `LocalRef` — the *address* of a local's slot (a borrow of the slot, not a read
  of its value); used to capture a `MUT` binding into a non-escaping callback
- `Global` — top-level binding references
- `FunctionRef` — references to named functions
- `Closure` — closure values with a name, type, and captured variable list
- `Capture` — reference to a captured variable by index inside a closure body
- `Call` — calls to functions that return a plain value
- `CallResult` — calls to functions that return `Result<T>`
- `Constructor` — record/struct constructors
- `UnionWrap` — wrapping a value as a specific union member
- `UnionExtract` — extracting the inner value from a union member
- `ResultIsOk` — testing whether a `Result` value is the success case
- `ResultValue` — extracting the success value from a `Result`
- `ResultError` — extracting the error value from a `Result`
- `WithUpdate` — record update expressions
- `ListLiteral` — list literal values
- `MapLiteral` — map literal values
- `MemberAccess` — member field access
- `Binary` — binary operator expressions
- `Unary` — unary operator expressions

`mfb build -ir` serializes this representation to `<project>.ir`.

IR is intentionally shared by both downstream products:

- Native executable generation lowers IR to target-specific native structures.
- Package generation lowers IR to architecture-independent MFPC binary representation.

## Lowering (AST → IR)

This section specifies the transforms `src/ir.rs` applies to turn the concrete,
syntaxchecked AST into the IR above. Lowering runs *after* syntaxcheck and
monomorphization, so every shape the algorithms rely on (inferred types, overload
selection, union membership) is already settled; the `expect("syntaxcheck
requires …")` panics throughout are assertions of that contract, not error
handling.

Source-level **semantics** belong to `./mfb spec language *`; the on-disk
**wire format** belongs to `./mfb spec package ir-section`. This topic owns only
the transform between them.

Two entry helpers carry the recursion:

- `lower_statement(stmt, locals, context, trap_name)` → `Vec<IrOp>` — one
  statement may expand to several ops (e.g. a counted `FOR` becomes three
  binds + a `For`).
- `lower_expression_with_expected(expr, expected, locals, context)` → `IrValue`,
  with `lower_expression` as the `expected = None` shorthand. The `expected` type
  flows downward to drive literal typing, union auto-wrap, and builtin default
  padding.[[src/ir/lower.rs:lower_statement]] [[src/ir/lower.rs:lower_expression_with_expected]]

`LowerContext` threads the mutable state: a `$`-prefixed temp counter
(`make_temp_local_name`), a lambda counter and accumulator, the current file
path and declared return type, the `recover_targets` stack, the set of `MUT`
local names, and the one-shot `nonescaping_callback` flag.[[src/ir/lower.rs:LowerContext]]

### 1. Built-in source-package injection (lowering entry)

`lower_project_with_external_functions` is the entry point. Before any lowering it
re-augments the AST by threading it through a **fixed ordered chain** of built-in
source packages: `json → csv → regex → datetime → vector → http → net → crypto → encoding`,
each via that module's `augmented_project`. The order is load-bearing: `http`
must precede `net` because `http_package.mfb` imports `net`, and `crypto` must
precede `encoding` for the same reason. Each step is `expect`-unwrapped
(the built-in package sources are compiler assets and must parse). Only after the
full chain runs are `function_returns`/`function_types`/`function_params`, the
`TypeIndex`, and the `LowerContext` built over the augmented
AST.[[src/ir/lower.rs:lower_project_with_external_functions]]

Per file, items lower as: `Function → lower_function`, `Type → lower_type`;
`Binding`s were pre-lowered (`lower_bindings`/`infer_binding_types`); `Resource`,
`FuncAlias`, `Link`, and `Doc` items carry no executable body and are surfaced
separately (native metadata, docs). Lifted lambdas (`context.lambdas`) are
appended to `functions` last.

### 2. Literal const typing

`Expression::Number` chooses its `Const` type from the **expected** type first:
`Fixed` if expected is `Fixed`, `Byte` if expected is `Byte`; otherwise by
`.`-presence — `Float` when the literal text contains `.`, else `Integer`.
`String`/`Boolean` map to their obvious `Const` types. The identifier `NOTHING`
lowers to a `Const { type_: "Nothing", value: "NOTHING" }`. A package constant
identifier (`builtins::is_package_constant`) is inlined to a `Const` carrying the
package-defined type and value. `literal_expression_type` is the type-only
companion used where no expected type is supplied (e.g. list-element
inference).[[src/ir/lower.rs:literal_expression_type]]

List/map literals: a `ListLiteral`'s element type comes from the expected
`List OF …` strip, else the first element's `literal_expression_type`, else
`Unknown`; each element is lowered with that expected element type. A
`MapLiteral`'s own type string is taken from the AST-supplied `key_type`/
`value_type` (set by syntaxcheck); the entries' keys/values are lowered with the
expected key/value types parsed from an expected `Map OF K TO V`.

### 3. Implicit union auto-wrap

`wrap_union_value(base, expr, expected, locals, context)` wraps a value whose
static type is a *variant* of an expected union into
`IrValue::UnionWrap { union_type, member_type, value }`, so the tag is explicit
in the IR (and faithfully serialized) rather than re-derived in codegen.
Membership is decided by `type_index.variant_belongs_to_union`. **Double-wrap is
avoided**: if `base` is already a `UnionWrap`, it is returned unchanged. The wrap
is applied at `LET`/`MUT` binds, at `RETURN` (against `current_return_type`),
inside record constructor lowering, and on plain identifier
uses.[[src/ir/lower.rs:wrap_union_value]]

### 4. EXIT / CONTINUE lowering map

`Statement::Exit` maps by target: `EXIT FOR/DO/WHILE` → `ExitLoop { kind }`;
`EXIT SUB` → `Return { value: None }`; `EXIT FUNC` → **no op** (empty vec — the
function's own `RETURN` flow handles it); `EXIT PROGRAM` → `ExitProgram { code }`
(the code expression is required by the parser). `CONTINUE` → `ContinueLoop { kind }`.[[src/ir/lower.rs:lower_statement]]

### 5. Counted FOR lowering

A `FOR` over `start`/`end`/`step` lowers to a numeric induction loop. The loop's
numeric type is **promoted** across all three operands by
`promote_loop_numeric_type_name`, which folds `numeric_binary_result_type("+", …)`
left-to-right (`start ⊕ end ⊕ step`). A missing `STEP` defaults to the literal
`1` as a `Const` of the loop type (`numeric_constant_for_type`).
`end` and `step` are **pre-evaluated** into temps `$for_end`/`$for_step` (so they
are computed once); an internal induction temp `$for_iter` drives `IrOp::For`.
Inside the body the user's loop variable is **re-bound** to the iterator value as
its first op, then the body block lowers. The `For` op carries an `IrSourceLoc`
for increment-overflow reporting, with **column 0** (only the header line is
known).[[src/ir/lower.rs:promote_loop_numeric_type_name]] [[src/ir/lower.rs:lower_statement]]

`FOR EACH` lowers more directly: the element type comes from
`collection_iteration_type` (§14), the loop var is bound to that type, and the
iterable lowers to `IrOp::ForEach`.

### 6. MATCH statement lowering

The scrutinee is lowered into a `$match` temp (`lower_match_expression` is just
`lower_expression_with_expected` against the matched type — a call scrutinee
auto-unwraps like any call; a value already of `Result OF …` type keeps that
shape). If the temp's type begins `Result OF `, an additional `$match_ok`
`Boolean` temp is bound to `ResultIsOk($match)` and **it** becomes the
`IrOp::Match` dispatch value; otherwise the scrutinee temp itself is the value.
Each case lowers to an `IrMatchCase { pattern, guard, body }` where `pattern` is
`IrMatchPattern::Else`, `::Value(v)`, or `::OneOf(vs)`. For a `Result` scrutinee a
`CASE Ok`/`CASE Error` union pattern lowers to a `Value` of the boolean constant
`true`/`false` (matching `$match_ok`); the `Ok`/`Error` binding, when present,
is bound at the case body's head via `ResultValue`/`ResultError`. A non-`Result`
union pattern lowers its `type_name` to a `Local` reference.[[src/ir/lower.rs:lower_match_expression]] [[src/ir/value.rs:IrMatchPattern]] [[src/ir/lower.rs:lower_match_case]]

### 7. Inline-TRAP desugaring

An inline `TRAP` (`… = <call> TRAP(e) <handler>` as the value of a `LET`/`MUT`,
an assignment, or a bare expression statement) is desugared in `lower_statement`
into existing primitives — **no backend op is emitted** for it. The three forms
become an `InlineTrapTarget`: `Bind { mutable, name, type_, explicit_type }`,
`Assign { name }`, or `Discard`.[[src/ir/lower.rs:InlineTrapTarget]]

`lower_inline_trap` produces, in order:[[src/ir/lower.rs:lower_inline_trap]]

```
Bind  $trap_res : Result OF T = <trapped expr, Call rewritten to CallResult>
Bind  $trap_val : T (mutable, uninitialized)        ; only for Bind/Assign targets
If ResultIsOk($trap_res):
  then:  $trap_val = ResultValue($trap_res)          ; empty for Discard
  else:  Bind <e> : Error = ResultError($trap_res)
         <treeified handler ops>                      ; RECOVER assigns into $trap_val
<target>                                              ; emitted once, after the If
```

Key points: the trapped `Call` is rewritten to `CallResult` so the raw `Result`
is observed instead of auto-unwrapped; the success value is delivered through the
shared `$trap_val` slot from **both** the Ok path and a `RECOVER`, so the final
target bind/assign is emitted exactly once after the `If`. The handler's error
name is bound to type `Error`. A `Discard` target allocates no `$trap_val` slot
and emits no trailing target. A `Bind` target whose `name` is mutable is also
recorded in `context.mutable_locals`; an `Assign` target emits `Assign` if the
name is a known local, else `AssignGlobal`.

### 8. Inline-TRAP handler `treeify` normalization

Because `RECOVER` lowers to a plain assignment that *falls through* to the
post-`If` target delivery, statements that textually follow a `RECOVER` must not
execute after recovery. `treeify_handler` rewrites the handler into a tree where
every leaf path ends in its own terminator, so the structured lowering needs no
jumps:[[src/ir/lower.rs:treeify_handler]]

- Statements after a **terminator** (`RETURN`/`EXIT`/`CONTINUE`/`FAIL`/
  `PROPAGATE`/`RECOVER`, per `statement_terminates`) are unreachable and
  **dropped**.
- When a fall-through `IF`/`MATCH` is followed by a continuation, the
  continuation is **distributed into each branch** (`distribute_continuation`):
  appended to any branch that does not already terminate, then re-treeified. A
  `MATCH` with no `ELSE` arm gains a **synthetic `ELSE`** carrying the
  continuation (an unmatched scrutinee otherwise falls through).
- A non-branching, non-terminating statement is kept and the tail continues to
  normalize.[[src/ir/lower.rs:distribute_continuation]]

`treeify_statement` recurses into nested blocks (loops, etc.) without
distributing a continuation, for the no-tail case.

### 9. RECOVER / PROPAGATE lowering

Neither has a dedicated op. A `RecoverTarget { slot, type_ }` stack
(`context.recover_targets`) is pushed by `lower_inline_trap` before lowering a
handler and popped after.[[src/ir/lower.rs:recover_targets]] [[src/ir/lower.rs:RecoverTarget]]

- `PROPAGATE` → `IrOp::Fail { error: Local(trap_name) }` — re-fails the current
  handler's bound error name (the `trap_name` argument threaded into
  `lower_statement`).
- `RECOVER value` → consults the innermost `RecoverTarget`: with a slot and a
  value, `Assign` into that slot; with a value but **no** slot (the `Discard`
  trap form), `Eval` the value; with no value, nothing. The recovered value is
  lowered with the target type as its expected type.[[src/ir/lower.rs:lower_statement]]

### 10. error() / Error / ErrorLoc construction

A call to `error(code, message)` — recognized only when `error` is **not**
shadowed by a user function (`function_params` lookup) — is lowered to nested
record constructors rather than a runtime call:
`Error(code, message, ErrorLoc(file, line, column))`. `build_error_value`
assembles the `Error` constructor; `error_loc_value` builds the `ErrorLoc` from
the current file plus the **call expression's** source location (`line`/`column`
of the `Call`). Thereafter the value flows as ordinary record
constructors.[[src/ir/lower.rs:build_error_value]] [[src/ir/lower.rs:error_loc_value]]

### 11. Source-location threading

`IrSourceLoc { line, column }` is attached to the IR nodes that can originate a
runtime error: `Call`, `CallResult` (helper-generated errors), `Binary`, `Unary`
(arithmetic overflow), and `For` (increment overflow). The file is **not** stored
per-node — it rides on `IrFunction::file` (the function's source file), and
nodes carry only line/column within it. Expression nodes take their loc from the
AST `line`/`column`; the `For` op uses the header line with **column 0**.
`error_loc_value` stamps `ErrorLoc` from the same loc + `current_file`.[[src/ir/types.rs:IrSourceLoc]]

### 12. Lambda lifting & closure / capture model

Each `Expression::Lambda` is **lifted** to a synthetic private `func` named
`$lambdaN` (monotonic counter), accumulated in `context.lambdas` and appended to
the project's functions. Its free variables are collected by `captured_locals`
(an AST walk that records outer locals not shadowed by the lambda's params,
deduplicated; it does **not** descend into nested lambdas). An
assignment-bodied lambda also captures its assignment target even if the target
never appears on the RHS.[[src/ir/lower.rs:captured_locals]] [[src/ir/lower.rs:CapturedLocal]]

In the lifted body, each capture is **prepended as a `Bind`** sourced from
`IrValue::Capture { index, type_, by_ref }`, where `index` is the env-slot
position (a `u32`, the width the package format encodes, so an index cannot
silently change value across a serialization round trip). The body itself: an ordinary lambda lowers to
`Return(Some(body))` with the body's inferred return type; an **assignment-bodied**
lambda lowers to `target = <body>` followed by `Return(None)`, and its declared
return type is `Nothing`.

At the call site the lambda value is either a bare `IrValue::FunctionRef`
(when there are no captures) or an `IrValue::Closure { name, type_, captures }`
carrying the captured environment values.

### 13. MUT slot-borrow (`by_ref`) capture

When a lambda is the argument in a **proven non-escaping callback position**, a
captured `MUT` local is captured by the **address of its slot** rather than by
value, so the callback observes and mutates the live binding. The gate is
`context.nonescaping_callback`, set per-argument from
`builtins::is_nonescaping_callback_arg(callee, index)` and consumed once by the
lambda lowering (reset immediately so a sibling non-lambda argument never
inherits it). A capture is `by_ref` iff that flag is set **and** the captured name
is in `context.mutable_locals`.[[src/ir/value.rs:LocalRef]] [[src/builtins/mod.rs:is_nonescaping_callback_arg]]

A `by_ref` capture differs in two places: the closure's env value for it is
`IrValue::LocalRef { name, type_ }` (the slot address, a borrow) instead of the
lowered identifier value; and the lifted body's `Bind` for it is **mutable** with
`Capture { …, by_ref: true }`, so reads and writes deref through the slot
pointer.

### 14. RES STATE & resource-iteration encoding

A `RES` binding's `STATE T` is carried inline in the lowered **type string** as
`<Type> STATE <T>` (e.g. `File STATE Cursor`), so codegen can default-initialize
and address the state payload while the bare resource type stays recognizable.
The same encoding applies to `RES` **parameters** via `lower_param`.[[src/ir/lower.rs:lower_param]]

`collection_iteration_type` computes the `FOR EACH` element type: `List OF X` →
`X`, but `List OF RES File` strips the `RES` so the loop var binds the **bare**
resource (`File`); `Map OF K TO V` → `MapEntry OF K TO V` (also `RES`-stripped on
the value). `StateAssign` (`resource.state = value`) lowers to `IrOp::StateAssign`,
replacing a `RES` binding's `STATE` payload; the value is lowered with the
binding's state type as expected.[[src/ir/lower.rs:collection_iteration_type]]

### 15. resource_owners is serialized (format v4)

Each `IrFunction` carries `resource_owners`, computed at lowering time by escape
analysis (`crate::escape::analyze_function(function).owners()`): a map from `RES`
binding name to `ResOwner::Local` (closed in its own scope) or
`ResOwner::Float(collection)` (ownership floats into an outer collection, §15.6).
`encode_function` writes name, visibility, kind, isolated, params, returns, body,
file, loc, **and** `resource_owners` — the map is emitted as a vector of
`(name, tag)` pairs sorted by binding name for a deterministic encoding
(`tag` 0 = `Local`, 1 = `Float` + the target string). `decode_function` reads it
back. It **must** ride the format: a decoded package has no AST to re-run escape
analysis on, so without it the package-path `RES`/ownership rules
(`ir::verify`) would false-reject every resource binding and resource codegen
would lose its close-obligation drain points. Lifted lambdas are created with an
empty `resource_owners`.[[src/ir/binary.rs:encode_function]] [[src/ir/binary.rs:decode_function]]

### 16. Builtin call-arg normalization & default padding

A builtin call's arguments are first run through
`normalize_builtin_call_arguments`. Then:[[src/ir/lower.rs:lower_expression_with_expected]]

- **`collections.filter`** with a bare identifier predicate rewrites that
  argument to a typed `IrValue::FunctionRef` (predicate type derived from the
  collection's element type).
- **Default padding** for optional trailing args: `tls`, `regex`, and `datetime`
  pad with synthesized scalar `Const`s; `http` pads similarly, except a `Map OF …`
  default (`http::read`/`write`'s `headers`) becomes an **empty `MapLiteral`**,
  not a scalar const; `crypto` likewise pads, but a `List OF …` default becomes
  an **empty `ListLiteral`** (the list analog of http's map case).
- **Overridable builtins** (`toString`/`len` over a built-in package value type,
  per `builtins::general::is_overridable`) route to that package's internal
  override helper (e.g. `toString(net::Url)` → an internalized
  `#net_urlToString`); user overrides were already rewritten to concrete symbols
  by the monomorphizer and need no routing here.
- **Target resolution**: `datetime::` is arity-aware (`implementation_name` keyed
  by `args.len()`); `json`/`csv`/`regex`/`net`/`http` resolve via their
  `implementation_name`, all internalized; migrated `collections::`/`strings::`
  members keep their **qualified dotted** target through to codegen.
- `thread_resource_plane_target` aliases `thread::transfer`/`accept` onto the
  proven `send`/`receive` runtime.

User-function calls take a separate path (`lower_local_call_arguments`); the
`expected` type for each builtin argument comes from `call_argument_expected_type`.

### 17. Native LINK ABI lowering into IR metadata

`LINK` blocks carry no executable body; their ABI surface is collected separately
into `link_functions` → `Vec<IrLinkFunction>`. Each native function records its
alias/name/library/symbol, wrapper params and return type, `return_resource`
flag, and the C-level surface: `abi_slots` (each `IrAbiSlot { name, ctype, is_out }`,
the `is_out` flag distinguishing `OUT` slots), `abi_return_name`/
`abi_return_ctype`, `CONST` pins resolved by `eval_link_const` (numbers,
unary `±`, booleans → 0/1, `NOTHING` → 0/NULL), and `SUCCESS_ON`/`RESULT`
mapped by `lower_link_expr` into an `IrLinkExpr` tree (the `Var`/`Int`/`Compare`/
`And`/`Or`/`Not` mini-language over the single native return variable). A `FREE`
block becomes `IrFree { slot, symbol }`. `native_resources` and `link_aliases`
(re-export aliases whose target is a `LINK` function) are collected by separate
passes.[[src/ir/lower.rs:eval_link_const]] [[src/ir/link.rs:IrLinkExpr]] [[src/ir/link.rs:IrLinkFunction]]

### 18. Project documentation collection

`collect_project_docs` walks `DOC` blocks into a `ProjectDocs { package, decls }`.
Only **exported** declarations are persisted (a non-exported decl is documented in
source but never compiled into the package). A callable `DOC` block selects the
overload it documents: the one whose param types match the header's declared
params (`function_param_types` vs normalized header types), else the first
overload of the matching kind (`FUNC`/`SUB`). Each `IrDocDecl` flattens the
block's prose (`(kind code, text)` pairs), args, props, return, errors, example,
`GROUP`, the `INTERNAL` attribute, and an optional `deprecated` message; the
package-level block becomes `IrPackageDoc`.[[src/ir/lower.rs:collect_project_docs]] [[src/ir/mod.rs:IrDocDecl]]
