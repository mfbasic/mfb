NAME
  errors - Error, Result, and TRAP behavior

SYNOPSIS
  mfb man errors

DESCRIPTION
  MFBASIC represents failures as values. Every FUNC and SUB has an implicit
  Result-shaped return: success is Ok(value), failure is Err(error). Ordinary
  calls auto-unwrap Ok values. If a call produces Err, the error transfers to
  the enclosing TRAP. If there is no TRAP, the error propagates to the caller.

  The standard error payload is:

    TYPE Error
      code AS Integer
      message AS String
    END TYPE

  Error codes are Integers. Program entry points that return an uncaught error
  write:

    Code: <code> Message: <message>

  to stderr and exit with status 255.

RESULT FLOW
  A call in an ordinary expression auto-unwraps:

    LET value = toInt(input)

  If toInt succeeds, value receives the Integer. If toInt fails, later work in
  the current expression is skipped and the Error routes to the nearest TRAP or
  propagates out of the function.

  Function arguments are evaluated left to right. If an earlier argument fails,
  later arguments are not evaluated.

  A call used as the direct scrutinee of MATCH is not auto-unwrapped. This is
  the local handling form:

    MATCH toInt(input)
      CASE Ok(value) : RETURN value
      CASE Err(err) : RETURN 0
    END MATCH

FAIL
  FAIL explicitly creates the current function's error result:

    IF n < 0 THEN FAIL error(77050002, "negative")

  With a TRAP, FAIL transfers to that TRAP. Without a TRAP, FAIL returns
  Err(error) to the caller.

TRAP
  Each FUNC or SUB may have at most one TRAP. It appears at the bottom of the
  function after normal flow. The TRAP payload is always Error:

    FUNC readAge(input AS String) AS Integer
      LET age = toInt(input)
      IF age < 0 THEN FAIL error(77050002, "negative")
      RETURN age

      TRAP err
        io::print("bad age: " & err.message)
        RETURN 0
      END TRAP
    END FUNC

  A TRAP is not reached by normal fall-through. Body paths before the TRAP must
  end with RETURN or FAIL.

TRAP OUTCOMES
  RETURN value
    Converts the error into the function's final success value. In a SUB, bare
    RETURN succeeds with NOTHING.

  PROPAGATE
    Returns the current Err(err) to the caller. PROPAGATE is valid only inside a
    TRAP.

  FAIL error
    Replaces or wraps the current error and returns Err(error) to the caller, or
    routes it through the same TRAP context when used in normal function body
    code.

  There is no resume operation in TRAP. Once control enters a TRAP, the failed
  expression is abandoned.

MATCH VS TRAP
  Use MATCH when a specific call's error is expected and local to that
  expression.

  Use TRAP when a function should have one central error policy for failures
  from multiple calls or from explicit FAIL statements.

ENTRY POINTS
  Executable entry points may handle errors with TRAP like any other function.
  If an entry point returns an uncaught Err(error), the runtime prints the code
  and message to stderr and exits with status 255.
