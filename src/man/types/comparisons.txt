NAME
  comparisons - comparison operators and operand rules

SYNOPSIS
  =  <>  <  >  <=  >=

DESCRIPTION
  MFBASIC comparison operators return Boolean.

  Equality operators:

    =   equal
    <>  not equal

  Ordering operators:

    <   less than
    >   greater than
    <=  less than or equal
    >=  greater than or equal

RULES
  = and <> accept numeric operands using the normal numeric promotion rules, or
  any two compatible comparable operands such as Boolean or String.

  <, >, <=, and >= accept two numeric operands or two String operands. Mixed
  String/numeric ordering is a compile-time type error.

  Numeric comparisons promote Byte, Integer, Float, and Fixed using the same
  rules as numeric arithmetic, but always produce Boolean.

  Two String operands are ordered lexicographically by Unicode scalar value: the
  strings are compared scalar by scalar, the first differing position decides,
  and a string that is a prefix of the other compares less. This order is
  deterministic and identical on every target. It is not a locale or human
  collation and is not grapheme-cluster aware; caseFold or normalize first when
  case-insensitive or locale-aware ordering is needed.

COMPARABLE AND ORDERABLE
  Comparable types (= and <>): Integer, Float, Fixed, Boolean, String, Byte,
  Nothing, enums, and records whose fields are all comparable.

  Orderable types (<, >, <=, >=): Integer, Float, Fixed, Byte, and String.
  Boolean, Nothing, enums, unions, and records are comparable but not orderable.
  collections::sort and collections::sortBy require an orderable element or key
  type.

EXAMPLES
  LET same AS Boolean = "Ada" = "Ada"
  LET ordered AS Boolean = 1.5 < 2
  LET textOrder AS Boolean = "Ada" < "Grace"
  LET mixed AS Boolean = 1 = toByte(1)
