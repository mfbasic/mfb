NAME
  setUnderline - enable or disable the underline text attribute on the terminal surface

SYNOPSIS
  term::setUnderline(enabled AS Boolean) AS Nothing

PACKAGE
  term

IMPORTS
  IMPORT term

DESCRIPTION
  Turns the underline text attribute on or off for subsequent text drawn through
  the term:: surface. When enabled is True the underline attribute is switched on;
  when it is False the attribute is switched off. The change takes effect
  immediately and persists until it is changed by another term::setUnderline call
  or reset to the default by term::on; it does not affect text already on the
  screen, only text drawn after the call returns.

  The underline attribute is independent of the foreground color, background color,
  and bold attribute: setting it leaves those unchanged. Underline is a boolean
  attribute, so calling term::setUnderline(True) twice in a row has the same effect
  as calling it once.

  Like the rest of the term:: surface other than term::on and term::isOn,
  term::setUnderline is gated behind TUI mode: it is a no-op while TUI mode is off,
  so a program must call term::on first for the attribute change to take effect.
  term::on resets the underline attribute back to off, so any state set with
  term::setUnderline is discarded the next time term::on is called.

  The function has no return value; its only effect is to update the current
  underline attribute. The attribute set here can be read back with
  term::getUnderline.

PARAMETERS
  enabled AS Boolean
    Whether the underline attribute should be on. True enables underlined text;
    False disables it.

RETURN VALUE
  AS Nothing
    Returns nothing. The function is invoked for its side effect of setting the
    current underline attribute used for subsequently drawn text.

ERRORS
  No errors.

EXAMPLES
  Draw underlined text on the TUI surface:

    IMPORT term
    IMPORT io

    term::on()
    term::setUnderline(True)
    io::print("this is underlined")
    term::setUnderline(False)
    io::print("this is not")
    term::off()

  Set the underline attribute and read it back:

    IMPORT term

    term::on()
    term::setUnderline(True)
    LET u AS Boolean = term::getUnderline()
    term::off()
