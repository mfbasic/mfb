NAME
  hideCursor - hide the terminal cursor on the TUI surface

SYNOPSIS
  term::hideCursor() AS Nothing

PACKAGE
  term

IMPORTS
  IMPORT term

DESCRIPTION
  Hides the terminal's text cursor on the term:: drawing surface. After
  term::hideCursor returns, the cursor is no longer drawn and stays hidden as the
  cursor position is changed with term::moveTo or advanced by drawing, until it is
  shown again with term::showCursor or the visibility is reset by term::on.

  Cursor visibility is a boolean state independent of the foreground color,
  background color, and text attributes: hiding the cursor leaves those unchanged.
  The call is idempotent, so calling term::hideCursor twice in a row has the same
  effect as calling it once. On the console backend the function emits the terminal
  sequence that turns the hardware cursor off; in app mode it makes the
  application's terminal view stop drawing its cursor.

  Like the rest of the term:: surface other than term::on and term::isOn,
  term::hideCursor is gated behind TUI mode: it is a no-op while TUI mode is off,
  so a program must call term::on first for the cursor to be hidden. term::on
  itself makes the cursor visible as part of resetting the surface to defaults, so
  any visibility set with term::hideCursor is discarded the next time term::on is
  called.

  The function takes no arguments and has no return value; its only effect is to
  hide the cursor.

PARAMETERS
  None.

RETURN VALUE
  AS Nothing
    Returns nothing. The function is invoked for its side effect of hiding the
    terminal cursor.

ERRORS
  No errors.

EXAMPLES
  Hide the cursor while drawing, then show it again:

    IMPORT term
    IMPORT io

    term::on()
    term::hideCursor()
    term::moveTo(0, 0)
    io::print("drawing without a blinking cursor")
    term::showCursor()
    term::off()

SEE ALSO
  term::showCursor, term::moveTo, term::on, term::off
