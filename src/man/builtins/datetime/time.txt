NAME
  time - validating time-of-day Time constructor

SYNOPSIS
  datetime::time(hour AS Integer, minute AS Integer, second AS Integer = 0, nanos AS Integer = 0) AS Time

PACKAGE
  datetime

IMPORTS
  IMPORT datetime

DESCRIPTION
  datetime::time builds a Time of day from its hour, minute, second, and
  sub-second components. A Time names a position within a single 24-hour day
  and carries no calendar date and no zone; pair it with a Date through
  datetime::civil to build a zoned DateTime.

  The constructor validates each component against its civil range before
  returning, and there is no normalization or wrap-around: an out-of-range
  component is an error, not silently carried into the next unit. hour must be
  in 0 .. 23, where 0 is midnight and 23 is the final hour of the day. minute
  and second must each be in 0 .. 59; the model has no leap seconds, so 60 is
  never a valid second. nanos is the sub-second remainder and must be in
  0 .. 999_999_999.

  second and nanos default to 0, so a two-argument call names the top of a
  minute and a three-argument call names the top of a second. Unlike
  datetime::instant and datetime::duration, time is not overloaded but a single
  signature with trailing defaults, so the defaults apply and you may omit
  second, or both second and nanos (see standard_package.md).

  time is pure: the same arguments always yield the same Time, and it has no
  side effects.

PARAMETERS
  hour AS Integer
    The hour of the day. Must be in 0 .. 23, where 0 is midnight and 23 is the
    last hour. Any value outside this range is an error.

  minute AS Integer
    The minute of the hour. Must be in 0 .. 59. Any value outside this range is
    an error.

  second AS Integer
    The second of the minute. Must be in 0 .. 59; there are no leap seconds, so
    60 is rejected. Defaults to 0 when omitted.

  nanos AS Integer
    The sub-second remainder in nanoseconds. Must be in 0 .. 999_999_999.
    Defaults to 0 when omitted.

RETURN VALUE
  AS Time
    A Time holding the validated hour, minute, second, and nanos. Returned only
    when all four components fall within their civil ranges.

ERRORS
  77050002 (ErrInvalidArgument)
  Raised when hour is outside 0 .. 23, when minute or second is outside
  0 .. 59, or when nanos is outside 0 .. 999_999_999.

EXAMPLES
  Construct a time at the top of a minute (second and nanos default to 0):

    IMPORT datetime

    LET t AS Time = datetime::time(9, 30)

  Construct a time with whole seconds:

    IMPORT datetime

    LET t AS Time = datetime::time(23, 59, 59)

  Combine a date and time into a zoned DateTime:

    IMPORT datetime

    LET d AS Date = datetime::date(2026, 6, 26)
    LET t AS Time = datetime::time(9, 30)
    LET dt AS DateTime = datetime::civil(d, t, datetime::utc())

  An out-of-range field raises ErrInvalidArgument:

    IMPORT datetime

    LET bad AS Time = datetime::time(24, 0)

SEE ALSO
  datetime::date, datetime::civil, datetime::instant, datetime::duration
