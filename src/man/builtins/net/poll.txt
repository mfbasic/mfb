NAME
  poll - test whether a socket has data ready to read

SYNOPSIS
  net::poll(sock AS Socket) AS Boolean
  net::poll(sock AS Socket, timeoutMs AS Integer) AS Boolean

PACKAGE
  net

IMPORTS
  IMPORT net

DESCRIPTION
  poll reports whether a connected Socket is ready to read. It returns TRUE when
  a subsequent net::read or net::readText can proceed without blocking, including
  when the peer has closed and a read would return end of stream, and FALSE when
  no data is available before the timeout elapses. The socket must still be open.

  The optional timeoutMs bounds how long poll waits for readiness, in
  milliseconds. When omitted it defaults to 0, making poll a non-blocking check
  that returns immediately. A positive timeoutMs waits up to that many
  milliseconds; the timeout must not be negative.

  poll inspects readiness only and never consumes data, so a TRUE result leaves
  the bytes in place for the next read. It is the readiness test for a single
  Socket; there is no list overload, so poll each socket individually.

OVERLOADS
  net::poll(sock AS Socket) AS Boolean
    Checks readiness immediately and returns without waiting.

  net::poll(sock AS Socket, timeoutMs AS Integer) AS Boolean
    Waits at most timeoutMs for the socket to become readable.

PARAMETERS
  sock AS Socket
    An open, connected Socket resource, as returned by net::connectTcp or
    net::accept. The socket is inspected for readiness only; it is not consumed
    and no data is read.

  timeoutMs AS Integer
    Optional. The maximum time, in milliseconds, to wait for the socket to
    become readable. When omitted it defaults to 0, making poll a non-blocking
    check that returns immediately. A positive value waits up to that many
    milliseconds; the value must not be negative.

RETURN VALUE
  AS Boolean
    TRUE when the socket is ready to read, meaning a subsequent net::read or
    net::readText can proceed without blocking — including when the peer has
    closed and a read would return end of stream. FALSE when no data became
    available before the timeout elapsed.

ERRORS
  77050002 (ErrInvalidArgument)
  Raised when timeoutMs is negative.

  77030004 (ErrResourceClosed)
  Raised when the socket has already been closed, or when the underlying
  readiness check on the socket fails.

EXAMPLES
  Check whether data is waiting without blocking:

    IMPORT net

    RES client = net::connectTcp("example.com", 80)
    IF net::poll(client) THEN
      LET response = net::readText(client, 1024)
    END IF
    ' client is closed by lexical drop when this scope ends

  Wait up to one second for a client to send a request:

    IMPORT net

    RES listener = net::listenTcp("0.0.0.0", 8080)
    RES client = net::accept(listener)
    IF net::poll(client, 1000) THEN
      LET request = net::readText(client, 1024)
    END IF
    ' client and listener are closed by lexical drop when this scope ends
