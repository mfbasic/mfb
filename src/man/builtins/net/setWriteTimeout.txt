NAME
  setWriteTimeout - bound how long a write on a socket may block

SYNOPSIS
  net::setWriteTimeout(sock AS Socket, timeoutMs AS Integer) AS Nothing
  net::setWriteTimeout(sock AS UdpSocket, timeoutMs AS Integer) AS Nothing

PACKAGE
  net

IMPORTS
  IMPORT net

DESCRIPTION
  setWriteTimeout sets the maximum time, in milliseconds, that a send on sock may
  block waiting for the underlying buffer to accept data. It applies to a
  connected TCP Socket or a bound UDP UdpSocket and takes effect on every
  subsequent write: net::write and net::writeText for a Socket, and net::sendTo
  and net::sendTextTo for a UdpSocket.

  When the timeout elapses before the write can make progress, the pending write
  fails with an Error rather than blocking further. The timeout governs only how
  long a single write waits for the buffer to drain; it does not cap the total
  time a loop of writes may take.

  A timeoutMs of 0 disables the timeout, so writes block indefinitely until the
  data is accepted, the peer closes the connection, or an error occurs. This is
  the default state of a freshly opened socket. timeoutMs must not be negative.

OVERLOADS
  net::setWriteTimeout(sock AS Socket, timeoutMs AS Integer) AS Nothing
    Bounds writes on a connected TCP socket.

  net::setWriteTimeout(sock AS UdpSocket, timeoutMs AS Integer) AS Nothing
    Bounds sends on a bound UDP socket.

PARAMETERS
  sock AS Socket
    An open, connected TCP Socket, as returned by net::connectTcp or
    net::accept, whose subsequent writes are to be bounded. The socket must still
    be open.

  sock AS UdpSocket
    An open, bound UdpSocket, as returned by net::bindUdp, whose subsequent sends
    are to be bounded. The socket must still be open.

  timeoutMs AS Integer
    The maximum time, in milliseconds, that a subsequent write or send may block
    waiting for the buffer to accept data. A value of 0 disables the timeout so
    writes block indefinitely, which is the default state of a freshly opened
    socket. A positive value bounds each write to that many milliseconds. The
    value must not be negative.

RETURN VALUE
  AS Nothing
    Returns nothing. On success the write timeout has been applied to sock and
    takes effect on every subsequent write.

ERRORS
  77050002 (ErrInvalidArgument)
  Raised when timeoutMs is negative.

  77030004 (ErrResourceClosed)
  Raised when sock has already been closed, or when the timeout could not be
  applied to the underlying socket handle (the setsockopt call failed).

EXAMPLES
  Fail a write that stalls for more than two seconds:

    IMPORT net

    RES client = net::connectTcp("example.com", 80)
    net::setWriteTimeout(client, 2000)
    net::writeText(client, "GET / HTTP/1.0\r\n\r\n")
    ' client is closed by lexical drop when this scope ends

  Bound a UDP send so a stalled buffer does not block forever:

    IMPORT collections
    IMPORT net

    RES sock = net::bindUdp("0.0.0.0", 0)
    net::setWriteTimeout(sock, 1000)
    LET dest = collections::get(net::lookup("127.0.0.1", 9000), 0)
    net::sendTextTo(sock, dest, "ping")
    ' sock is closed by lexical drop when this scope ends
