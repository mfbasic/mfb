NAME
  readText - read available bytes from a connected socket as UTF-8 text

SYNOPSIS
  net::readText(sock AS Socket, maxBytes AS Integer) AS String

PACKAGE
  net

IMPORTS
  IMPORT net

DESCRIPTION
  Receives data from a connected Socket and returns it decoded as a UTF-8
  String. A single call performs one underlying receive: it returns as soon as
  any data is available rather than waiting to fill the requested size, so the
  returned String is frequently shorter than maxBytes bytes. The socket must
  still be open.

  The call blocks until at least one byte arrives, the peer closes its side of
  the connection, or the socket's read timeout elapses. maxBytes bounds the
  number of bytes read in this call; it does not request that exactly that many
  bytes be read. On success the returned String is built from at least one byte.

  Like net::read, this function does not signal end of stream with an empty
  result: when the peer has closed the connection it raises an
  ErrConnectionClosed error. To consume a whole message, call readText in a
  loop, appending each result, and stop when an ErrConnectionClosed error is
  raised. Use net::poll to test for readiness without blocking, and
  net::setReadTimeout to bound how long a read may wait.

  The received bytes are validated as UTF-8 before being returned; invalid
  UTF-8 raises an ErrEncoding error. Because a single receive may split a
  multi-byte UTF-8 sequence across calls, use net::read instead when the peer
  sends raw binary data, or when you need to reassemble bytes spanning multiple
  reads before decoding.

  The data is decoded into a freshly allocated String; the function has no
  other side effects and does not close the socket.

PARAMETERS
  sock AS Socket
    A connected socket to receive from, such as one returned by net::connectTcp
    or net::accept. It must still be open; reading from a closed socket is an
    error.

  maxBytes AS Integer
    The maximum number of bytes to read in this call. Must be positive. It caps
    the number of bytes received before decoding but does not guarantee that
    many bytes are returned.

RETURN VALUE
  AS String
    The bytes received in this read, decoded as UTF-8, in the order they
    arrived. The String is built from between 1 and maxBytes bytes inclusive.
    End of stream is not reported as an empty String; it is reported as an
    ErrConnectionClosed error.

ERRORS
  77050002 (ErrInvalidArgument)
  Raised when maxBytes is not positive.

  77030004 (ErrResourceClosed)
  Raised when sock has already been closed.

  77070004 (ErrConnectionClosed)
  Raised when the peer has closed the connection (an end-of-stream read) or the
  read otherwise fails for a reason other than a timeout.

  77070005 (ErrReadTimeout)
  Raised when the socket's read timeout elapses before any data arrives.

  77020004 (ErrEncoding)
  Raised when the received bytes are not valid UTF-8.

  77010001 (ErrOutOfMemory)
  Raised when the temporary read buffer or the returned String cannot be
  allocated.

EXAMPLES
  Read up to 1024 bytes of text from a connected socket:

    IMPORT net

    RES client = net::connectTcp("example.com", 80)
    net::writeText(client, "GET / HTTP/1.0\r\n\r\n")
    LET reply = net::readText(client, 1024)
    ' client is closed by lexical drop when this scope ends

  Drain a connection until the peer closes it:

    IMPORT net

    RES client = net::connectTcp("example.com", 80)
    net::writeText(client, "GET / HTTP/1.0\r\n\r\n")
    MUT response AS String = ""
    DO
      MATCH net::readText(client, 4096)
        CASE Ok(chunk) : response = response & chunk
        CASE Err(err) : EXIT DO
      END MATCH
    LOOP
    ' client is closed by lexical drop when this scope ends
