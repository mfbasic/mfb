NAME
  receiveTextFrom - receive a single UDP datagram as UTF-8 text with its sender address

SYNOPSIS
  net::receiveTextFrom(sock AS UdpSocket, maxBytes AS Integer) AS DatagramText

PACKAGE
  net

IMPORTS
  IMPORT net

DESCRIPTION
  Receives exactly one datagram from a UDP socket and returns it as a
  DatagramText record with two fields: from, the sender's Address, and value,
  the payload decoded as a UTF-8 String. Unlike a connected stream, each call
  returns one whole datagram and reports who sent it, so the same bound socket
  can receive from many peers without establishing a connection.

  A datagram is delivered whole or not at all. maxBytes bounds the payload that
  will be accepted: if the incoming payload is larger than maxBytes the call
  raises ErrMessageTooLarge rather than truncating it, so size maxBytes to the
  largest message you expect. The returned String holds the entire payload and
  is frequently shorter than maxBytes bytes; a zero-length payload yields an
  empty String rather than an error.

  The payload bytes are validated as UTF-8 before being returned; invalid UTF-8
  raises an ErrEncoding error. Because a datagram is received whole, a UTF-8
  sequence is never split across calls; use net::receiveFrom instead when the
  payload is raw binary data and a List OF Byte is more appropriate than a
  String.

  The call blocks until a datagram arrives or the socket's read timeout elapses.
  Use net::setReadTimeout to bound how long a receive may wait; when the timeout
  expires the call raises ErrReadTimeout. Bind the socket with net::bindUdp and
  send datagrams with net::sendTextTo.

  The payload is decoded into a freshly allocated String and the sender Address
  into a freshly allocated record; the function has no other side effects and
  does not close the socket.

PARAMETERS
  sock AS UdpSocket
    A bound UDP socket to receive from, such as one returned by net::bindUdp. It
    must still be open; receiving on a closed socket is an error.

  maxBytes AS Integer
    The maximum payload size, in bytes, that will be accepted. Must be positive.
    A datagram whose payload exceeds maxBytes is rejected with ErrMessageTooLarge
    rather than truncated.

RETURN VALUE
  AS DatagramText
    A DatagramText record with field from set to the sender's Address and field
    value set to the payload decoded as a UTF-8 String. The String is built from
    between 0 and maxBytes bytes inclusive and holds the datagram's entire
    payload.

ERRORS
  77050002 (ErrInvalidArgument)
  Raised when maxBytes is not positive.

  77030004 (ErrResourceClosed)
  Raised when sock has already been closed.

  77070007 (ErrMessageTooLarge)
  Raised when the received datagram's payload does not fit in maxBytes.

  77070005 (ErrReadTimeout)
  Raised when the socket's read timeout elapses before a datagram arrives.

  77020004 (ErrEncoding)
  Raised when the received payload is not valid UTF-8.

  77070001 (ErrAddressInvalid)
  Raised when the sender's address cannot be represented as an Address.

  77070003 (ErrNetworkFailed)
  Raised when the receive fails for a reason other than a timeout.

  77010001 (ErrOutOfMemory)
  Raised when the receive buffer, the returned String, or the sender Address
  record cannot be allocated.

EXAMPLES
  Receive one text datagram and read its payload and sender:

    IMPORT net

    RES sock = net::bindUdp("0.0.0.0", 9000)
    LET dg = net::receiveTextFrom(sock, 1024)
    ' dg.value holds the text payload; dg.from holds the sender Address
    ' sock is closed by lexical drop when this scope ends

  Bound how long a receive waits, then handle a timeout:

    IMPORT net

    RES sock = net::bindUdp("127.0.0.1", 0)
    net::setReadTimeout(sock, 2000)
    LET dg = net::receiveTextFrom(sock, 512)
    io::print(dg.value)
    TRAP(e)
      ' e.code is 77070005 (ErrReadTimeout) when no datagram arrived in time
      io::print(toString(e.code))
    END TRAP
