NAME
  openFile - open a file resource

SYNOPSIS
  fs::openFile(path AS String, mode AS String = "read") AS File

DESCRIPTION
  Opens path and returns an opaque File resource. The mode argument is optional
  and defaults to "read". Portable modes are "read" or "r", "write" or "w",
  "readWrite" or "rw", and "append" or "a".

  Unlike fs::open, mode may be omitted to open the file for reading.

  File resources are closed by lexical drop at scope exit or explicitly with
  fs::close.

ERRORS
  Raises an Error when mode is not one of the portable modes, when path does not
  exist for a read mode, when the containing directory is missing or not
  writable for a write or append mode, or when the host OS otherwise refuses to
  open the file.

EXAMPLES
  RES f = fs::openFile("data.txt")
  ' f is closed by lexical drop when this scope ends

  RES w = fs::openFile("out.txt", "write")
  ' opened for writing

SEE ALSO
  fs::open, fs::openFileNoFollow, fs::close
