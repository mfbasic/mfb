NAME
  isNumeric - test whether text can be parsed as a number

SYNOPSIS
  isNumeric(value AS String) AS Boolean

PACKAGE
  general

IMPORTS
  None. general functions are available without an IMPORT statement.

DESCRIPTION
  isNumeric returns TRUE when value can be parsed as an Integer, Float, or Fixed
  value by the standard conversion functions.

  Non-finite numeric names such as NaN or Infinity are not valid successful
  numeric values in ordinary MFBASIC source.

OVERLOADS
  isNumeric(value AS String) AS Boolean
    Tests whether value is accepted by a numeric parser.

ERRORS
  isNumeric returns FALSE for invalid numeric text instead of raising a parse
  error. Errors can still propagate from evaluating value before isNumeric is
  called.

TYPE CHECKING
  isNumeric accepts only String values.

EXAMPLES
  Guard a parse:

    IF isNumeric("42") THEN
      LET value AS Integer = toInt("42")
    END IF

SEE ALSO
  toInt
  toFloat
  toFixed

Values like very large exponent text can be accepted by `isNumeric` even though the documented “standard conversion functions” would reject overflow/non-finite numeric results.
- Inconsistency 2: `src/arch/aarch64_old/mod.rs` has a separate `isNumeric` implementation that accepts only optional `-` followed by digits. That conflicts with the manual/spec behavior and the dedicated valid test cases for decimal/exponent forms such as `.5`, `-2.50`, and `1.25e2`.

Notes: Resolve by making `isNumeric` share the same successful-parse criteria as the numeric string conversions, including rejecting overflow/non-finite results. Remove, update, or prove unreachable the old AArch64 implementation so every `src/**` path recognizes the same grammar.
