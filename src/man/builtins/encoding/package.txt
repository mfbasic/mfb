NAME
  encoding - byte<->text and Unicode codecs

SYNOPSIS
  IMPORT encoding
  encoding::hexEncode(data)
  encoding::base64Encode(data)
  encoding::utf8Encode(value)
  encoding::percentEncode(text)
  encoding::punycodeEncode(domain)

DESCRIPTION
  The encoding package converts between raw bytes and text and between text and
  Unicode code units. Raw List OF Byte values (hashes, MACs, keys, random bytes,
  binary file contents) are unusable as text without these codecs. encoding is a
  built-in package implemented in MFBASIC source over the bits, strings, and
  collections packages: IMPORT encoding needs no manifest dependency.

  Encoders are total. Decoders fail closed with ErrInvalidFormat (77050003) on
  malformed input, so a TRAP can recover from bad data. Outputs are standardized
  (RFC 4648 for hex/Base32/Base64/Base64url, RFC 3986 for percent-escaping,
  RFC 3492 for Punycode), so the native and Binary Representation execution paths
  produce identical results.

  utf8Encode is a return-type overload: it produces a List OF Byte (the raw bytes
  binary and crypto consumers need) or a List OF Integer (the same 0..255 values
  as Integers, for arithmetic on code units), selected by the call's expected
  type. An unannotated call is a TYPE_OVERLOAD_AMBIGUOUS compile error; supply the
  type (for example LET raw AS List OF Byte = encoding::utf8Encode(s)). utf8Decode
  is selected by its parameter type (List OF Byte or List OF Integer).

FUNCTIONS
  utf8Encode, utf8Decode      String <-> UTF-8 bytes
  utf16Encode, utf16Decode    String <-> UTF-16 code units (numeric)
  utf32Encode, utf32Decode    String <-> Unicode scalar values (numeric)
  hexEncode, hexDecode        bytes <-> lowercase hex
  base32Encode, base32Decode  bytes <-> Base32 (RFC 4648 6)
  base64Encode, base64Decode  bytes <-> standard Base64 (RFC 4648 4)
  base64UrlEncode, base64UrlDecode   bytes <-> URL-safe Base64 (RFC 4648 5)
  percentEncode, percentDecode       String <-> percent-escaped (RFC 3986)
  formUrlEncode, formUrlDecode       String <-> application/x-www-form-urlencoded
  htmlEscape, htmlUnescape    String <-> HTML entity-escaped text
  punycodeEncode, punycodeDecode     hostname <-> ASCII Punycode (RFC 3492)
  uleb128Encode, uleb128Decode       Integer <-> unsigned LEB128 bytes
  sleb128Encode, sleb128Decode       Integer <-> signed LEB128 bytes
  varintEncode, varintDecode  Integer <-> ZigZag + LEB128 bytes

ERRORS
  77050003 (ErrInvalidFormat)
  Raised by the decode functions on malformed input (a non-hex digit, an invalid
  Base64 alphabet/length/padding, a malformed %XX escape or bad UTF-8, an
  out-of-range or surrogate code unit, malformed Punycode, or LEB128 overflow or
  truncation), and by uleb128Encode when given a negative value.
