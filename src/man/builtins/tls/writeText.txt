NAME
  writeText - write a String to a connected TLS socket as UTF-8 text

SYNOPSIS
  tls::writeText(sock AS TlsSocket, value AS String) AS Nothing

PACKAGE
  tls

IMPORTS
  IMPORT tls

DESCRIPTION
  writeText sends the UTF-8 bytes of value as encrypted application data over a
  connected TlsSocket and returns nothing. The String's packed byte data is
  written directly: a String already holds well-formed UTF-8, so the bytes are
  written exactly as held, with no re-encoding, decoding, or newline
  translation. The socket must still be open.

  writeText writes the entire String before returning. It loops internally,
  advancing past the bytes that each underlying TLS write accepted and reissuing
  the write for the remainder, so a partial TLS write that transfers only some of
  the buffer is resumed rather than treated as complete. When the call returns
  successfully, every byte of value has been handed to the TLS layer for
  transmission; this does not guarantee the peer has received or read them. An
  empty String writes nothing and returns immediately.

  Unlike net::writeText, which distinguishes a closed peer from a write timeout,
  this function maps every underlying TLS write failure -- a peer that closed the
  session, a reset connection, or any other SSL_write error -- to ErrTlsFailed.
  Use tls::write instead when sending raw binary data from a List OF Byte rather
  than UTF-8 text from a String.

  TLS is implemented on Linux by driving the system OpenSSL library
  (libssl.so.3, falling back to libssl.so.1.1); if neither library can be loaded
  or a required symbol is missing, writeText raises ErrTlsFailed. On macOS the
  send is performed through Network.framework, and a failed send likewise raises
  ErrTlsFailed. The function does not allocate, reorder, or close the socket, and
  has no side effect beyond transmitting the data.

PARAMETERS
  sock AS TlsSocket
    A connected TLS socket to send on, as returned by tls::connect. It must
    still be open; writing to a closed socket is an error.

  value AS String
    The text to send. Written verbatim as the String's UTF-8 bytes, in order. An
    empty String writes nothing and returns immediately.

RETURN VALUE
  AS Nothing
    Returns nothing on success, after every byte of value has been handed to the
    TLS layer for transmission.

ERRORS
  77030004 (ErrResourceClosed)
  Raised when sock has already been closed.

  77070008 (ErrTlsFailed)
  Raised when the underlying TLS write fails for any reason -- including a peer
  that has closed the session or a reset connection -- or when the system
  OpenSSL library or a required symbol could not be loaded.

EXAMPLES
  Send an HTTP request as text over a connected TLS socket:

    IMPORT tls

    RES conn = tls::connect("example.com", 443)
    tls::writeText(conn, "GET / HTTP/1.0" + Chr(13) + Chr(10) + Chr(13) + Chr(10))
    LET reply = tls::readText(conn, 4096)
    ' conn is closed by lexical drop when this scope ends
