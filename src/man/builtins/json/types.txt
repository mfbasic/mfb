NAME
  types - the json package record types

SYNOPSIS
  json::JsonNull
  json::JsonBool
  json::JsonNum
  json::JsonStr
  json::JsonArr
  json::JsonObj

PACKAGE
  json

IMPORTS
  IMPORT json

  json is a built-in package, so IMPORT json needs no manifest dependency.

DESCRIPTION
  The json package represents a JSON value as a tree of these six record types.
  Each record wraps a single field carrying one JSON form. The Json union ranges
  over all six, so every JSON value parses to exactly one of these variants and
  json::stringify accepts either the Json union or any one member type directly.

  JsonNull, JsonBool, JsonNum, and JsonStr are leaf scalars. JsonArr and JsonObj
  are containers whose fields hold further Json values, which is what lets a Json
  value nest to arbitrary depth.

TYPES
  json::JsonNull
    The JSON literal null.

    value AS Nothing
      Always the absent value Nothing; the record carries no further data and
      exists only to mark the null form.

  json::JsonBool
    A JSON boolean (true or false).

    value AS Boolean
      The boolean payload: True for the JSON literal true, False for false.

  json::JsonNum
    A JSON number.

    value AS Float
      The numeric payload, carried as a Float. Because JSON numbers are stored
      as Float, very large or very precise integers may lose precision in a
      parse/stringify round trip, and a non-finite value (NaN or infinity) has
      no JSON form and fails on json::stringify.

  json::JsonStr
    A JSON string.

    value AS String
      The decoded string contents, with JSON escapes already resolved; on
      stringify the standard JSON string escapes are reapplied.

  json::JsonArr
    A JSON array.

    items AS List OF Json
      The array elements in document order, each a nested Json value. Order is
      preserved by json::stringify.

  json::JsonObj
    A JSON object.

    fields AS Map OF String TO Json
      The object members keyed by name, each value a nested Json value. The path
      readers json::get and json::getOr descend through these maps, and
      json::stringify emits the pairs in the map's iteration order.
