NAME
  json - JSON parsing, formatting, and object-path helpers

SYNOPSIS
  IMPORT json
  json::parse(value)
  json::stringify(value)
  json::get(value, path)
  json::getOr(value, path, defaultValue)

DESCRIPTION
  The json package provides a built-in Json union plus helpers for parsing
  UTF-8 JSON text, formatting Json values, and reading nested object members by
  key path.

  The built-in Json type is:

    UNION Json
      Null(value AS Nothing)
      Bool(value AS Boolean)
      Num(value AS Float)
      Str(value AS String)
      Arr(items AS List OF Json)
      Obj(fields AS Map OF String TO Json)
    END UNION

  json::get and json::getOr traverse only object members. Path elements are
  String keys. Arrays are represented as Json values but are not indexed by
  these helpers.

FIXME
- Inconsistency 1: `json::parse` is documented/spec’d as parsing JSON text, but `src/builtins/json_package.mfb` accepts unescaped control characters inside strings because `__json_parseString` appends any non-quote, non-backslash character. JSON requires U+0000 through U+001F to be escaped.
- Inconsistency 2: `json::stringify` is spec’d to produce compact valid JSON text, but `__json_escapeString` only escapes selected controls (`\b`, `\f`, `\n`, `\r`, `\t`) and can emit other raw control characters, producing invalid JSON.

Notes: Reject raw control characters in `__json_parseString`. Update `__json_escapeString` to escape all U+0000 through U+001F, likely as `\u00XX` for controls without short escapes.
