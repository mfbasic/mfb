NAME
  transfer - move a resource across a thread boundary

SYNOPSIS
  thread::transfer(t AS Thread OF RES Res TO Out, res AS RES Res, timeoutMs AS Integer = 0) AS Nothing
  thread::transfer(t AS ThreadWorker OF RES Res TO Out, res AS RES Res, timeoutMs AS Integer = 0) AS Nothing

PACKAGE
  thread

IMPORTS
  IMPORT thread

DESCRIPTION
  Moves a thread-sendable resource across the thread boundary on the resource
  plane, the dedicated queue selected by the handle type. A parent Thread handle
  transfers the resource onto the worker's inbound resource queue, so it is
  delivered to the worker; a ThreadWorker handle transfers onto the
  parent-visible outbound resource queue, so it is delivered back to the parent.
  The matching reader is thread::accept on the opposite handle. The resource
  plane carries resources only; plain values move on the separate data plane via
  thread::send, which is kept resource-free.

  transfer MOVES the resource: the sender's RES binding is consumed and cannot be
  used again on a successful path, transferring ownership to the receiving side.
  On a failed path no transfer happens and ownership returns to the sender, so a
  TRAP handler may reuse the binding.

  res must be a resource whose type matches the thread's resource type Res (a
  thread whose resource type is Unknown accepts any resource). Transferring a
  non-resource value fails to type-check; use thread::send for data.

  timeoutMs controls how long the call waits for queue space when the destination
  resource queue is already full. With timeoutMs = 0 (the default) the call does
  not wait: it enqueues immediately if there is room and otherwise fails at once
  with ErrTimeout. A positive timeoutMs waits up to that many milliseconds for
  space before failing with ErrTimeout. A negative timeoutMs is rejected with
  ErrInvalidArgument. When there is room in the queue, the resource is enqueued
  immediately regardless of timeoutMs and the call returns Nothing.

  The call participates in cooperative cancellation. A transfer that is blocked
  waiting for queue space wakes and fails with ErrInterrupted when the thread has
  ended or cancellation has been requested, or when the destination queue has
  been closed. On the parent Thread handle, a transfer against a thread handle
  that has already been closed (for example after its result was retrieved with
  thread::waitFor) fails with ErrResourceClosed.

OVERLOADS
  thread::transfer(t AS Thread OF RES Res TO Out, res AS RES Res, timeoutMs AS Integer = 0) AS Nothing
    Parent-side transfer. Moves the resource onto the worker's inbound resource
    queue. May fail with ErrResourceClosed when the parent Thread handle is
    already closed, and with ErrInterrupted when the worker has completed or
    cancellation was requested.

  thread::transfer(t AS ThreadWorker OF RES Res TO Out, res AS RES Res, timeoutMs AS Integer = 0) AS Nothing
    Worker-side transfer. Moves the resource onto the parent-visible outbound
    resource queue. A worker-side transfer blocked on queue space wakes and fails
    with ErrInterrupted when cancellation of the worker is requested or the queue
    is closed.

PARAMETERS
  t AS Thread OF RES Res TO Out | ThreadWorker OF RES Res TO Out
    The thread handle whose resource-plane queue receives the resource. A parent
    Thread handle targets the worker inbound resource queue; a ThreadWorker handle
    targets the outbound resource queue read by the parent.

  res AS RES Res
    The resource to move. Must match the thread's resource type Res; a thread
    typed with resource Unknown accepts any resource. The RES binding is consumed
    on success and ownership passes to the receiving side; on failure the binding
    remains usable by the sender.

  timeoutMs AS Integer
    Maximum time in milliseconds to wait for space when the destination resource
    queue is full. 0 (the default) does not wait and fails immediately with
    ErrTimeout if the queue is full. A positive value waits up to that many
    milliseconds. A negative value is rejected with ErrInvalidArgument.

RETURN VALUE
  AS Nothing
    Returns Nothing once the resource has been enqueued onto the resource plane.

ERRORS
  77050002 (ErrInvalidArgument)
  Raised when timeoutMs is negative.

  77050008 (ErrTimeout)
  Raised when the destination resource queue is full and space does not become
  available before timeoutMs elapses (immediately when timeoutMs is 0).

  77050009 (ErrInterrupted)
  Raised when a waiting or attempted transfer observes that the thread has ended
  or cancellation has been requested, or that the destination queue has been
  closed.

  77030004 (ErrResourceClosed)
  Raised by the parent Thread overload when the Thread handle has already been
  closed, such as after its result was retrieved with thread::waitFor or t.result.

TYPE CHECKING
  res must be a resource type assignable to the handle's resource type Res. A
  handle whose resource type is Unknown accepts any resource; otherwise res's
  static type must equal Res. A non-resource value, or a thread handle that
  carries no resource plane, fails to type-check. timeoutMs must be an Integer.
  The result type is always Nothing.

EXAMPLES
  Move a file resource to the worker without waiting for queue space:

    RES f AS File = fs::openFile("data.txt")
    thread::transfer(t, f)

  Wait up to 10 milliseconds for room on the resource queue:

    thread::transfer(t, f, 10)
