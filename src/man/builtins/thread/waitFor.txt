NAME
  waitFor - wait for a worker result

SYNOPSIS
  thread::waitFor(t AS Thread OF Msg TO Out) AS Out

DESCRIPTION
  Waits until the worker has stored its result, then returns the successful Out
  value. Err results auto-propagate using the normal fallible function rules.

EXAMPLES
  LET count AS Integer = thread::waitFor(t)

FIXME

- Inconsistency 2: The specs state `thread::waitFor` may be called before or after draining outbound messages and returns the stored result; the manual only says it waits for the worker result and does not document the queue-draining interaction.
- Inconsistency 3: The manual/spec document `Err` auto-propagation from the stored worker result, but the targeted tests do not cover a worker returning/failing with `Err` and `waitFor` propagating it.
