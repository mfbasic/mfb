NAME
  thread - start isolated workers and exchange bounded queue messages

SYNOPSIS
  IMPORT thread
  thread::start(worker, data)
  thread::waitFor(t)

DESCRIPTION
  The thread package starts exported ISOLATED FUNC entry points from imported
  packages in fresh package instances. Parent code holds Thread values. Worker
  code holds ThreadWorker values. Together they carry an inbound message queue,
  an outbound message queue, cancellation state, and the worker result.
  Retrieving the result with thread::waitFor(t) or t.result is one-shot and
  closes the parent Thread handle.

  Thread values are non-copyable owned handles. A live parent Thread is cleaned
  up automatically on scope exit, RETURN, FAIL, propagated errors, trap routing,
  and successful reassignment of a MUT Thread. Dropping a running Thread
  requests cancellation, wakes its queues, and detaches the worker. A Thread
  moved out by RETURN or another consuming operation is not dropped by the
  source scope. Compiler-generated cleanup is idempotent for handles already
  closed by thread::waitFor(t) or t.result.

  There is intentionally no thread::stop(). Cancellation is cooperative:
  thread::cancel(t) sets a flag, and the worker observes it with
  thread::isCancelled(t). Runtime-managed worker queue waits inside the worker,
  including thread::receive(ThreadWorker, ...) and
  thread::send(ThreadWorker, ...), wake and fail with ErrInterrupted when
  cancellation is requested. Other blocking built-ins that are implemented as
  runtime-managed waits must use the same cooperative error-return model when
  cancellation integration is provided. Cancellation does not asynchronously
  kill a worker while it owns a resource handle, holds a queue lock, moves a
  non-copyable value, writes its result, or runs package/native code.

  There is also no separate thread::detach() source API. Dropping a running
  Thread already requests cancellation and detaches the OS worker for eventual
  runtime cleanup. An explicit detach operation would have the same ownership
  and cleanup requirements while making abandoned workers easier to create.

  Worker entry points must have the shape:

    ISOLATED FUNC(ThreadWorker OF Msg TO Out, In) AS Out

  The entry point must be an exported function from an imported package.
  Lambdas, closures, SUBs, non-isolated functions, current-package functions,
  and functions without the leading ThreadWorker parameter are rejected at
  compile time.
