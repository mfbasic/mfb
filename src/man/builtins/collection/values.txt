NAME
  values - return the values from a map

SYNOPSIS
  values OF K, V(value AS Map OF K TO V) AS List OF V

PACKAGE
  collection

IMPORTS
  None. collection functions are available without an IMPORT statement.

DESCRIPTION
  values returns the values from value in key iteration order. The result list
  contains one item for each map entry.

OVERLOADS
  values OF K, V(value AS Map OF K TO V) AS List OF V
    Returns the map values.

ERRORS
  values does not fail for an evaluated map. Errors can still propagate from
  evaluating value before values is called.

TYPE CHECKING
  values accepts only Map OF K TO V values.

EXAMPLES
  Get map values:

    LET scores AS List OF Integer = values(Map OF String TO Integer { "a" := 1 })

SEE ALSO
  keys
  len

FIXME
- Inconsistency 1: The manual says `values` “does not fail for an evaluated map,” but native lowering allocates a new list for the projection and has an allocation-error return path. Runtime allocation failure is therefore possible.
