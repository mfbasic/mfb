REQUIREMENT: Decimal conversion exponent overflow
SPECIFICATION: standard_package.md @ §3.1 General Built-ins
GROUP: stdlib
RECOMMENDED: CODE CHANGE
VERIFIED: YES
PRIORITY: MED

=== io ===

REQUIREMENT: `io::flush` and `io::input` must flush standard output streams, not fsync file descriptors
SPECIFICATION: standard_package.md @ §7 Built-in IO Package
GROUP: io
RECOMMENDED: CODE CHANGE
VERIFIED: YES
PRIORITY: MED
`io::flush` and `io::flushError` are specified to flush standard output/error, and `io::input` must flush standard output after writing a non-empty prompt. The native helpers lower these flushes to `fsync(1)`/`fsync(2)`, which is durable file synchronization, not stream flushing. Because `io::print`/`io::write` use raw `write` calls rather than a buffered stream, there may be no userspace buffer to flush, and `fsync` can fail on valid standard streams such as terminals or pipes. Resolve by removing `fsync` from standard-stream flush semantics: either make flush a successful no-op for the current unbuffered raw-write implementation, or introduce an actual runtime output buffer and flush that buffer with `write`. Update `io::input` prompt flushing accordingly and add runtime tests for flush/input with stdout attached to a terminal, pipe, and regular file.
