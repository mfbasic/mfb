REQUIREMENT: Direct calls through first-class function values
SPECIFICATION: specifications/mfbasic.md @ §6 Functions
The spec requires first-class function values to be callable, including examples like `RETURN f(f(x))`. The typechecker accepts calls through local bindings or parameters whose type is `FUNC(...) AS ...`, and bytecode lowering has an `OPCODE_CALL_VALUE_RESULT` path, but the active shared native lowering treats `NirValue::Call { target: "f" }` as a direct symbol call. For a function-valued local or parameter, there is no native indirect-call lowering that loads the callable value and branches through it, so valid first-class function programs can typecheck but fail or link incorrectly in native builds. Resolve by preserving call-target kind in IR/NIR for local function-value calls, lowering those calls with the callable address loaded from the local/parameter slot and `blr`, applying normal argument/result auto-unwrapping semantics, and adding runtime tests for function parameters, lambda values, nested calls such as `f(f(x))`, and package/native paths.
